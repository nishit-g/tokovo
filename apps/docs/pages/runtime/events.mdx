# Runtime Events

import { Callout } from 'nextra/components'

<Callout type="info">
<strong>Events are commands.</strong> Each event tells the runtime what happens at a specific frame. The engine processes them to build world state.
</Callout>

Events are the **execution contract** between the compiler and runtime.

---

## Event Structure

```typescript
interface TimelineEvent {
  at: number;      // Frame when this happens
  kind: EventKind; // Category
  type: string;    // Specific type
  _trace?: Trace;  // Debug info
  // ... event-specific fields
}
```

## Event Kinds

| Kind | Description | Examples |
|------|-------------|----------|
| `DEVICE` | Device state changes | UNLOCK, LOCK, OPEN_APP, SHOW_NOTIFICATION |
| `APP` | App-specific actions | MESSAGE_RECEIVED, TYPING_START, MESSAGE_STATUS |
| `CAMERA` | Camera commands | ZOOM, PAN, SHAKE, FOCUS, RESET, ANCHOR_FOCUS |
| `AUDIO` | Audio events | PLAY_SOUND, STOP_SOUND, FADE_VOLUME |
| `KEYBOARD` | Virtual keyboard | SHOW, HIDE, TYPE_CHAR, BACKSPACE, CLEAR |
| `OS` | Device OS state | SET_TIME, SET_BATTERY, SET_NETWORK, SET_DND |
| `TOUCH` | Touch gestures | TAP, LONG_PRESS, DRAG, SCROLL |
| `CALL` | Phone calls | INCOMING, ANSWER, DECLINE, END, TOGGLE_MUTE |

## Event Ordering

Events are sorted by:

1. `at` (frame)
2. `phase` (DEVICE < NAV < APP < FX)
3. `priority` (within phase)
4. `trackId` (for concurrent operations)
5. `opIndex` (stable tie-breaker)

## Event Traces

Every event includes trace info for debugging:

```typescript
interface Trace {
  episodeId: string;
  beat: string;
  trackId: number;
  opIndex: number;
  phase: number;
  priority: number;
}
```
