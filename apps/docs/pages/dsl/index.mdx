# DSL Overview

import { Callout, Tabs, Tab } from 'nextra/components'

<Callout type="info">
<strong>Two APIs for different needs.</strong> The Author DSL is for semantic storytelling. Event Factories are for frame-precise control.
</Callout>

The Tokovo DSL is a **TypeScript fluent API** for writing stories that compile to deterministic video.

## Two APIs

| API | Package Path | Purpose | Target User |
|-----|-------------|---------|-------------|
| **Author DSL** | `@tokovo/dsl/author` | High-level, semantic | Content creators, AI |
| **Event Factories** | `@tokovo/dsl/events` | Frame-precise control | Developers, debugging |

---

## Author DSL (High-Level)

The Author DSL lets you express **what happens** without worrying about frames or timing.

### Design Goals

1. **Human-readable** — Writers can understand it
2. **AI-friendly** — Structured, deterministic output
3. **Zero frames** — Express intent, not timing
4. **Fully typed** — IDE autocomplete + compile-time safety

### Basic Structure

```typescript
import { episode } from "@tokovo/dsl";

const scene = episode("breakup-01", ep => {
  // Episode configuration
  ep.config({ 
    pacing: "slow-burn",
    director: "auto" 
  });

  // Device context
  ep.device("AlicePhone", "iphone16", d => {
    d.app("app_whatsapp");
    d.conversation("dm_bob", { name: "Bob" });

    // Beats (semantic groups)
    d.beat("tension", b => {
      b.wait("2s");
      b.typing("Bob").for("1.5s");
      b.receive("Bob", "We need to talk.");
    });
  });
});
```

### Hierarchy

```
episode()
  └── ep.device()
        ├── d.app()
        ├── d.conversation()
        └── d.beat()
              ├── b.wait()
              ├── b.typing().for()
              ├── b.send()
              ├── b.receive()
              ├── b.read()
              ├── b.delete()
              ├── b.concurrent()
              ├── b.pov()
              └── b.screenshot()
```

### Output: Scene IR

The Author DSL produces **Scene IR** — semantic truth with no frames:

```typescript
// Scene IR output
{
  episodeId: "breakup-01",
  meta: { fps: 30, pacing: "slow-burn" },
  devices: [{
    deviceId: "AlicePhone",
    profileId: "iphone16",
    beats: [{
      name: "tension",
      ops: [
        { kind: "Wait", duration: "2s" },
        { kind: "TypingStart", actor: "Bob", conversationId: "dm_bob" },
        { kind: "Wait", duration: "1.5s" },
        { kind: "TypingEnd", actor: "Bob", conversationId: "dm_bob" },
        { kind: "ReceiveMessage", actor: "Bob", text: "We need to talk.", ... }
      ]
    }]
  }]
}
```

## Event Factories (Low-Level)

Event Factories give you **frame-precise control** over timeline events. Use these when you need exact timing.

```typescript
import { dsl } from "@tokovo/dsl";

const events: TimelineEvent[] = [
  // Messages
  dsl.messages.receive(0, "conv_1", "Alice", "Hello!"),
  dsl.messages.send(30, "conv_1", "Hi there!"),
  dsl.messages.typingStart(60, "conv_1", "Alice"),
  dsl.messages.typingEnd(90, "conv_1", "Alice"),
  
  // Semantic Camera (RECOMMENDED)
  dsl.camera.anchorFocus(30, "lastMessage", "dramatic", 5),
  dsl.camera.anchorTrack(60, "inputArea", 40, 0.18, "subtle"),
  dsl.camera.hold(100, 18),
  ...dsl.camera.punchGlide(120, "lastMessage"),
  dsl.camera.reset(180, 25),
  
  // Keyboard
  dsl.keyboard.show(0, "phone"),
  dsl.keyboard.typeChar(10, "phone", "H"),
  dsl.keyboard.keyDown(10, "phone", "H"),
  dsl.keyboard.keyUp(12, "phone"),
  dsl.keyboard.hide(60, "phone"),
  
  // Audio
  dsl.audio.play(30, "notification.mp3", { bus: "ui", volume: 0.8 }),
  
  // Device OS
  dsl.os.setTime(0, Date.now()),
  dsl.os.setBattery(0, 87),
];
```

### Available Modules

| Module | Events | Notes |
|--------|--------|-------|
| `dsl.messages` | `receive`, `send`, `typingStart`, `typingEnd`, `markSent`, `markDelivered`, `markRead` | Core messaging |
| `dsl.camera` | `anchorFocus`, `anchorTrack`, `hold`, `punchGlide`, `zoom`, `shake`, `reset` | Semantic + legacy |
| `dsl.keyboard` | `show`, `hide`, `typeChar`, `backspace`, `keyDown`, `keyUp`, `clear`, `setText` | Virtual keyboard |
| `dsl.audio` | `play`, `stop`, `fade`, `backgroundMusic` | Sound effects |
| `dsl.os` | `setTime`, `setBattery`, `setNetwork`, `setDND`, `setAirplane` | Device state |
| `dsl.touch` | `tap`, `longPress`, `drag`, `scroll` | Touch gestures |
| `dsl.call` | `incoming`, `answer`, `decline`, `end`, `hold` | Phone calls |

## Camera Events (Semantic)

<Callout type="warning">
<strong>Prefer semantic camera events.</strong> Use `anchorFocus` and `anchorTrack` instead of raw `zoom` with pixel coordinates.
</Callout>

### anchorFocus

One-time focus on a semantic anchor:

```typescript
dsl.camera.anchorFocus(
  frame,          // Start frame
  "lastMessage",  // Semantic anchor
  "dramatic",     // Shot preset
  5               // Optional shake intensity
)
```

### anchorTrack

Continuous following with smoothing (webseries camera operator feel):

```typescript
dsl.camera.anchorTrack(
  frame,          // Start frame
  "lastMessage",  // Anchor to track
  40,             // Duration in frames
  0.18,           // Smoothing (0.08=slow, 0.18=operator, 0.6=whip)
  "operatorFollow" // Shot preset
)
```

### punchGlide

Signature webseries move: punch in + smooth glide. Returns TWO events:

```typescript
const [punch, glide] = dsl.camera.punchGlide(frame, "lastMessage");
// punch: ANCHOR_FOCUS with impact preset (10f)
// glide: ANCHOR_TRACK with operatorFollow (35f)

// Use spread in your events array:
const events = [
  ...dsl.camera.punchGlide(100, "lastMessage"),
  // More events...
];
```

### hold

Intentional stillness for reading:

```typescript
dsl.camera.hold(frame, 18)  // Hold for 18 frames (minimum 12 recommended)
```

### reset

Return to neutral:

```typescript
dsl.camera.reset(frame, 25)  // Duration in frames
```

## Typing Humanizer

Generate realistic typing with variable speed and optional typos:

```typescript
import { generateTyping, TYPING_SPEEDS } from "@tokovo/dsl";

// Generate realistic typing
const events = generateTyping(0, "phone", "Hello world!", {
  speed: "casual",        // "slow" | "casual" | "fast" | "angry"
  typoPositions: [5],     // Optional: positions that get typos
  typoRate: 0.05,         // Optional: random typo rate (0-1)
  seed: 12345,            // Optional: deterministic randomness
});
```

### Typing Speeds

| Speed | Frames/Char | Chars/Sec | Use Case |
|-------|-------------|-----------|----------|
| `slow` | 10 | 3 | Thoughtful typing |
| `casual` | 5 | 6 | Normal conversation |
| `fast` | 3 | 10 | Rapid replies |
| `angry` | 2 | 15 | Urgent/emotional |

## When to Use Which API

<Tabs items={['Author DSL', 'Event Factories']}>
  <Tab>
    **Use Author DSL when:**
    - Writing conversational stories
    - AI is generating content
    - You want semantic understanding
    - Pacing should be automatic
    - You don't care about exact frame timing
    
    ```typescript
    // Perfect for: "Bob and Alice have a fight"
    d.beat("confrontation", b => {
      b.receive("Bob", "We need to talk.");
      b.pause("2s");
      b.typing("Alice").for("3s");
      b.send("What's wrong?");
    });
    ```
  </Tab>
  <Tab>
    **Use Event Factories when:**
    - Building showcase demos
    - Debugging camera behavior
    - Precise timing is critical
    - Integrating with custom systems
    - You need frame-by-frame control
    
    ```typescript
    // Perfect for: "At exactly frame 60, do this"
    const events = [
      dsl.messages.receive(60, "conv", "Bob", "Hi!"),
      dsl.camera.anchorFocus(60, "lastMessage", "impact", 5),
      dsl.camera.hold(85, 18),
      dsl.camera.reset(103, 25),
    ];
    ```
  </Tab>
</Tabs>
