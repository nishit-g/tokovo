# Renderer Engines

The renderer uses a **three-engine architecture** that separates computation from rendering.

## Why Engines?

| Problem | Solution |
|---------|----------|
| **God component** | TokovoRenderer was 350+ lines doing everything |
| **Slow iteration** | New templates required touching camera, layout, overlays |
| **Hard debugging** | If a bubble is 5px wrong, which layer is guilty? |
| **Cascading bugs** | Camera bug could kill chat UI |

## Architecture

```
┌─────────────────┐     ┌─────────────────┐
│ useLayoutEngine │ ──► │ useCameraEngine │
│  world + t      │     │  + layout       │
│  → layout       │     │  → transform    │
└─────────────────┘     └─────────────────┘
        │                       │
        └───────┬───────────────┘
                ▼
        ┌───────────────┐
        │ TokovoRenderer│ (dumb paint)
        └───────────────┘
                │
┌───────────────┴───────────────┐
│         useAudioEngine        │
│  world + t → busStates        │
└───────────────────────────────┘
```

**Mental Model**: *Engines build blueprints. Renderer paints blueprints.*

## Layout Engine

Determines **what to show** and **where**.

```typescript
import { useLayoutEngine } from "@tokovo/renderer";

const layoutOutput = useLayoutEngine({
  world,
  t,
  focusDeviceId: "phone",
});

// Output
layoutOutput.viewKind;      // "CHAT" | "FEED" | "LOCKSCREEN" | ...
layoutOutput.layout;        // LayoutState (positions, scrollY)
layoutOutput.profile;       // DeviceProfile (iPhone16, Pixel)
layoutOutput.variant;       // "ios" | "android"
layoutOutput.isError;       // false (or true if device missing)
```

### LayoutEngineOutput

```typescript
interface LayoutEngineOutput {
  deviceId: string;
  device: DeviceState;
  appId: string | undefined;
  viewKind: ViewKind;
  layout: LayoutState;           // Use layout.kind to narrow
  profile: DeviceProfile;
  variant: "ios" | "android";
  activeConversationId?: string;
  effectiveViewportHeight: number;
  isError: boolean;
}
```

### Error Handling

If the device is missing, the engine returns `NULL_LAYOUT_OUTPUT`:

```typescript
if (layoutOutput.isError) {
  // Render black screen, not crash
  return <div style={{ backgroundColor: "#000" }}>Device not found</div>;
}
```

## Camera Engine

Determines **camera transform** (zoom, pan, shake).

```typescript
import { useCameraEngine } from "@tokovo/renderer";

const cameraOutput = useCameraEngine({
  world,
  t,
  layoutOutput,
  eventIndex,         // For DirectorLite
  directorEnabled: true,
});

// Output
cameraOutput.transform;     // CameraTransform
cameraOutput.cameraStyle;   // React.CSSProperties (ready to apply)
cameraOutput.deviceStyle;   // Legacy layout transforms
cameraOutput.directorOutput; // DirectorLite debug info
```

### CameraEngineOutput

```typescript
interface CameraEngineOutput {
  transform: CameraTransform;
  cameraStyle: React.CSSProperties;
  deviceStyle: React.CSSProperties;
  directorOutput?: DirectorOutput;
}
```

### DirectorLite Integration

When `directorEnabled` is true and `eventIndex` is provided:

1. Scans events in window (past 90 frames, future 15 frames)
2. Extracts signals (NewMessage, TypingStarted, etc.)
3. Derives camera effects (ZoomToRect, PushIn, MicroShake)
4. Applies effects to produce final transform

## Audio Engine

Determines **what sounds to play** and **at what volume**.

```typescript
import { useAudioEngine } from "@tokovo/renderer";

const audioOutput = useAudioEngine({
  world,
  t,
  focusDeviceId: "phone",
});

// Output
audioOutput.busStates;        // { sfx: { duckMultiplier: 0.8 }, ... }
audioOutput.activeSounds;     // Array of [id, sound] tuples
audioOutput.musicBed;         // Current music (if any)
audioOutput.musicCrossfade;   // { outVolume, inVolume }
audioOutput.hasAudio;         // true if audio state exists
```

### AudioEngineOutput

```typescript
interface AudioEngineOutput {
  busStates: Record<string, BusState>;
  activeSounds: Array<[string, SoundCue | ActiveSound]>;
  musicBed: MusicBed | null;
  musicCrossfade: { outVolume: number; inVolume: number };
  backgroundMusic: unknown | null;
  hasAudio: boolean;
}
```

## TokovoRenderer (Dumb Paint)

After engines compute, the renderer just paints:

```tsx
const TokovoRenderer: React.FC<Props> = (props) => {
  // 1. Get blueprints
  const layoutOutput = useLayoutEngine({ world, t, focusDeviceId });
  const cameraOutput = useCameraEngine({ world, t, layoutOutput, eventIndex });

  // 2. Paint
  return (
    <div style={{ width: profile.width, height: profile.height }}>
      <div style={cameraOutput.cameraStyle}>
        <DeviceFrame>
          <AppView layout={layoutOutput.layout} />
          <Overlays />
        </DeviceFrame>
      </div>
    </div>
  );
};
```

No compute. No branching. Just wiring + paint.

## Memoization Reality

These hooks run every frame because `world` changes each frame:

```typescript
// world reference changes → useMemo cache invalidated
useLayoutEngine({ world, t });
```

**This is intentional.** The engines are the Update Loop. Keep computations fast:
- No object allocations in loops
- No deep cloning
- No expensive traversals

## Adding New Formats

Before: Touch TokovoRenderer, break everything.

After:
1. Add new layout strategy in `layout/strategies/`
2. Add `viewKind` case in `useLayoutEngine`
3. Camera and Audio keep working

New format = **mostly layout changes**.

## Debugging

| Symptom | Guilty Engine |
|---------|---------------|
| Wrong position/scroll | Layout Engine |
| Wrong zoom/pan/shake | Camera Engine |
| Wrong volume/ducking | Audio Engine |
| Visual glitch | TokovoRenderer (CSS) |

Each engine is isolated. Changes don't cascade.
