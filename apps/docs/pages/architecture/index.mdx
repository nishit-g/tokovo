# Architecture Overview

import { Callout, Cards, Card } from 'nextra/components'

<Callout type="info">
<strong>Mental Model</strong>: Stories flow down through layers. Pixels flow up through engines.
</Callout>

Tokovo is built on a **strict layered architecture** that enforces separation of concerns.

---

## The Four Layers

```
┌───────────────────────────┐
│        AUTHOR DSL         │  ← Writers / AI / GUI
│  (TypeScript fluent API)  │
└────────────┬──────────────┘
             ↓
┌────────────▼──────────────┐
│        SCENE IR           │  ← Semantic intent (no frames)
│  (beats, actions, waits)  │
└────────────┬──────────────┘
             ↓
┌────────────▼──────────────┐
│       TIMELINE IR         │  ← Frame-based execution
│   (at=frame, payload)     │
└────────────┬──────────────┘
             ↓
┌────────────▼──────────────┐
│      RENDERER ENGINES     │  ← Layout/Camera/Audio
│     + DUMB RENDERER       │  ← Paints pixels
└───────────────────────────┘
```

---

## The 3-Engine System

<Callout type="warning">
<strong>The renderer doesn't compute.</strong> All logic lives in three engines that produce blueprints.
</Callout>

```
WorldState + Frame
        │
        ▼
┌───────────────────────────────────────────┐
│            TokovoRenderer (Orchestrator)   │
├───────────────────────────────────────────┤
│                                           │
│   useLayoutEngine()                       │
│        │ → ViewKind, LayoutState,         │
│        │   DeviceProfile, Platform        │
│        ▼                                  │
│   useCameraEngine()                       │
│        │ → CameraTransform, CSS Styles    │
│        │   (uses layout for anchors)      │
│        ▼                                  │
│   useAudioEngine()                        │
│        │ → BusStates, ActiveSounds,       │
│        │   MusicBed, Ducking              │
│                                           │
├───────────────────────────────────────────┤
│   JSX Output → DeviceFrame, AppView,      │
│                Overlays, Notifications    │
└───────────────────────────────────────────┘
```

### Engine Responsibilities

| Engine | Input | Output | Responsibility |
|--------|-------|--------|---------------|
| **Layout** | `world`, `t`, `focusDeviceId` | `LayoutState`, `ViewKind` | What to show, where |
| **Camera** | `world`, `t`, `layoutOutput` | `CameraTransform`, CSS | Zoom, pan, shake |
| **Audio** | `world`, `t`, `focusDeviceId` | `BusStates`, sounds | Volume, ducking |

### Why This Split?

| Symptom Before | Solution Now |
|----------------|--------------|
| 350+ line god component | Each engine isolated |
| Break everything on change | Changes don't cascade |
| "Which layer is buggy?" | Clear ownership |
| Can't test camera alone | Each engine unit-testable |

## Key Invariants

### 1. DSL Never Imports Runtime

```
@tokovo/dsl      → @tokovo/ir        ✅
@tokovo/compiler → @tokovo/ir        ✅
@tokovo/core     → @tokovo/dsl       ❌ NEVER
```

### 2. Scene IR Has No Frames

Scene IR captures **WHAT HAPPENS**, not **WHEN**.

```typescript
// Scene IR (no frames)
{ kind: "Wait", duration: "1.5s" }
{ kind: "ReceiveMessage", actor: "Bob", text: "Hi" }

// Timeline IR (has frames)
{ at: 45, kind: "MessageReceived", ... }
```

### 3. Compiler is Pure

Every compiler pass is a **pure function**:

```typescript
function normalize(sceneIR: SceneIR): SceneIR { ... }
function resolveRefs(sceneIR: SceneIR): SceneIR { ... }
function timeLowering(sceneOps: SceneOp[]): TimelineOp[] { ... }
```

No side effects. No state. Same input → same output.

### 4. Engines Don't Mutate

Engines receive `WorldState` and return new objects. They never mutate input.

```typescript
// Engine pattern
function useLayoutEngine(input: LayoutEngineInput): LayoutEngineOutput {
  // Pure computation
  return { ... };
}
```

## Package Dependencies

```
@tokovo/ir        ← Pure types (no deps)
      ↑
@tokovo/dsl       ← Depends only on IR
      ↑
@tokovo/compiler  ← Depends only on IR
      
@tokovo/core      ← Runtime (isolated)
      ↑
@tokovo/renderer  ← Depends on core
      ↑
@tokovo/apps-*    ← Depends on core + renderer
      ↑
@tokovo/devices   ← Depends on core
```

## Data Flow

```
User writes DSL
       │
       ▼
episode() → SceneIR
       │
       ▼
compile() → TimelineIR → Events[]
       │
       ▼
replay(events, t) → WorldState
       │
       ▼
TokovoRenderer(world, t) → JSX → Remotion → Video
```

---

## Plugin System

Apps are self-contained plugins:

```
@tokovo/apps-whatsapp
├── plugin.ts        ← Registers with PluginManager
├── runtime.ts       ← Reducer (mutates WorldState)
├── ui.tsx           ← React view component
├── behaviors.ts     ← Camera intent mappings
├── anchor-provider.ts ← Semantic anchor extraction
└── notification-adapter.ts
```

See [Plugin System](/architecture/plugins) for details.

---

## Why This Matters

| Benefit | How |
|---------|-----|
| **AI generation** | Structured IR, not magic prompts |
| **Debugging** | Trace any frame to its beat |
| **Testing** | Golden tests on deterministic output |
| **FPS changes** | Scene IR unchanged |
| **Platform ports** | Same IR, different adapters |
| **Extensibility** | Add apps without modifying core |
