# Plugin System

import { Callout, Tabs, Tab, Steps, Cards, Card } from 'nextra/components'

<Callout type="info">
**Apps are plugins.** Every Tokovo app (WhatsApp, Phone, Twitter) is a self-contained plugin that owns its UI, state, sounds, camera behaviors, and widgets.
</Callout>

Tokovo uses a **plugin architecture** for apps, enabling you to add new apps without modifying the core engine.

## Why Plugins?

| Benefit | How |
|---------|-----|
| **Self-contained** | Each app owns UI, state, sounds, camera, widgets |
| **Auto-registration** | Plugins register at import time |
| **Extensible** | Add new apps without modifying core |
| **Typed** | Full TypeScript support with interfaces |
| **Isolated** | Bugs in one app don't crash others |

---

## 7 Registries

Tokovo uses **seven registries** to manage app components. Understanding these is key to building apps.

| Registry | Purpose | Package | Auto-registers? |
|----------|---------|---------|-----------------|
| `PluginManager` | App plugins | `@tokovo/core` | On `PluginManager.register()` |
| `ReducerRegistry` | State reducers | `@tokovo/core` | ✅ When plugin has `reducer` |
| `AnchorRegistry` | Semantic anchors | `@tokovo/core` | Manual |
| `WidgetRegistry` | Platform widgets | `@tokovo/core` | ✅ When plugin has `widgets` |
| `BehaviorRegistry` | Camera behaviors | `@tokovo/core` | Manual |
| `NotificationAdapterRegistry` | Notification formatting | `@tokovo/core` | Manual |
| `DeviceProfileRegistry` | Device profiles | `@tokovo/devices` | On import |

## Plugin Interface

The complete `TokovoPlugin` interface:

```typescript
interface TokovoPlugin {
  // === Identity ===
  id: string;                    // "app_whatsapp" (must start with app_)
  name: string;                  // "WhatsApp"
  version: string;               // "1.0.0"

  // === Branding ===
  icon?: string;                 // Icon path or emoji
  primaryColor?: string;         // "#25D366"

  // === Core Components ===
  appView?: AppViewComponent;    // React component for app UI
  reducer?: AppReducer;          // State reducer for events
  defaultState?: any;            // Initial app state

  // === Event Handling ===
  eventTypes?: string[];         // Events this plugin handles

  // === Audio ===
  sounds?: Record<string, string>;  // Sound mappings
  notificationSound?: string;       // Default notification sound

  // === Widgets (Dynamic Island, etc.) ===
  widgets?: WidgetSlot[];

  // === Notifications ===
  notificationAdapter?: NotificationAdapter;
}
```

## Adding a New App

<Steps>

### Step 1: Create Package

```bash
# Create package structure
mkdir -p packages/apps-myapp/src
cd packages/apps-myapp

# Create package.json
cat > package.json << 'EOF'
{
  "name": "@tokovo/apps-myapp",
  "version": "1.0.0",
  "main": "src/index.ts",
  "dependencies": {
    "@tokovo/core": "workspace:*"
  }
}
EOF
```

### Step 2: Define Plugin

```typescript
// packages/apps-myapp/src/plugin.ts
import { definePlugin, PluginManager, TokovoPlugin, AppViewComponent } from "@tokovo/core";
import { myAppReducer } from "./runtime";
import { MyAppView } from "./ui";

export const MyAppPlugin: TokovoPlugin = definePlugin({
  id: "app_myapp",
  name: "My App",
  version: "1.0.0",

  // Branding
  icon: "🚀",
  primaryColor: "#FF5722",

  // Core
  appView: MyAppView as unknown as AppViewComponent,
  reducer: myAppReducer,

  // Events this plugin handles
  eventTypes: [
    "MY_ACTION",
    "ANOTHER_ACTION",
  ],

  // Sounds
  sounds: {
    "action": "myapp-action.mp3",
    "notification": "myapp-notification.mp3",
  },
  notificationSound: "myapp-notification.mp3",
});

// Auto-register on import
PluginManager.register(MyAppPlugin);
```

### Step 3: Create Reducer

```typescript
// packages/apps-myapp/src/runtime.ts
import { WorldState, TimelineEvent, ReducerRegistry } from "@tokovo/core";
import produce from "immer";

export function myAppReducer(draft: WorldState, event: TimelineEvent): void {
  // Only handle APP events for our app
  if (event.kind !== "APP") return;
  if ((event as any).appId !== "app_myapp") return;

  switch (event.type) {
    case "MY_ACTION":
      // Mutate draft directly (uses Immer)
      if (!draft.appState.app_myapp) {
        draft.appState.app_myapp = { items: [] };
      }
      draft.appState.app_myapp.items.push((event as any).payload);
      break;

    case "SET_SCREEN":
      draft.appState.app_myapp.screen = (event as any).screen;
      break;
  }
}

// Register with engine (also done automatically via PluginManager)
ReducerRegistry.registerAppReducer("app_myapp", myAppReducer);
```

### Step 4: Create View Component

```tsx
// packages/apps-myapp/src/ui.tsx
import React from "react";
import { AppViewProps, getTokens, Platform } from "@tokovo/core";

export const MyAppView: React.FC<AppViewProps> = ({
  world,
  deviceId,
  platform = "ios",
  t = 0,
}) => {
  const tokens = getTokens(platform as Platform);
  const appState = world.appState.app_myapp || { items: [] };
  const device = world.devices[deviceId || Object.keys(world.devices)[0]];

  return (
    <div style={{
      width: "100%",
      height: "100%",
      backgroundColor: "#ffffff",
      fontFamily: tokens.fontFamily,
      display: "flex",
      flexDirection: "column",
    }}>
      {/* Header */}
      <header style={{
        height: 144,
        backgroundColor: "#FF5722",
        color: "#fff",
        display: "flex",
        alignItems: "center",
        padding: "0 36px",
        fontSize: 51,
        fontWeight: 600,
      }}>
        My App
      </header>

      {/* Content */}
      <main style={{ flex: 1, padding: 36 }}>
        {appState.items.map((item: any, i: number) => (
          <div key={i} style={{ marginBottom: 24 }}>{item}</div>
        ))}
      </main>
    </div>
  );
};
```

### Step 5: Define Camera Behaviors


### Step 6: Create Anchor Provider

```typescript
// packages/apps-myapp/src/anchor-provider.ts
import { AnchorProvider, AnchorSnapshot, AnchorRegistry, LayoutRect } from "@tokovo/core";

export const MyAppAnchorProvider: AnchorProvider = {
  appId: "app_myapp",

  getAnchors(world, layout, deviceId): AnchorSnapshot {
    const anchors: Record<string, LayoutRect> = {};

    // Extract semantic regions from your layout
    if (layout.kind === "MY_VIEW") {
      if (layout.mainContentRect) {
        anchors.mainContent = layout.mainContentRect;
      }
      if (layout.headerRect) {
        anchors.header = layout.headerRect;
      }
    }

    // Always provide device fallback
    anchors.device = { x: 0, y: 0, width: 430, height: 932 };

    return { anchors, deviceId, appId: "app_myapp" };
  }
};

// Register at module load
AnchorRegistry.register(MyAppAnchorProvider);
```

### Step 7: Create Notification Adapter (Optional)

```typescript
// packages/apps-myapp/src/notification-adapter.ts
import {
  NotificationAdapter,
  NotificationAdapterRegistry,
  Notification,
  FormattedNotification
} from "@tokovo/core";

const myAppAdapter: NotificationAdapter = {
  appId: "app_myapp",

  format(notification: Notification): FormattedNotification {
    return {
      title: notification.title,
      body: notification.body,
      icon: "🚀",
      iconBackground: "#FF5722",
      accentColor: "#FF5722",
      preview: notification.preview,
      actions: [
        { id: "open", label: "Open" },
        { id: "dismiss", label: "Dismiss" },
      ],
    };
  },

  handleAction(actionId: string, notification: Notification) {
    if (actionId === "open") {
      return [{
        at: Date.now(),
        kind: "DEVICE",
        deviceId: notification.deviceId || "phone",
        type: "OPEN_APP",
        appId: "app_myapp",
      }];
    }
    return [];
  },

  measureHeight(notification: Notification, viewport: { width: number }) {
    return 180;  // Base height in 3x scale
  },
};

NotificationAdapterRegistry.register(myAppAdapter);
```

## PluginManager API

```typescript
import { PluginManager } from "@tokovo/core";

// Register a plugin
PluginManager.register(MyPlugin);

// Get a plugin
const plugin = PluginManager.get("app_myapp");

// Get view component
const View = PluginManager.getView("app_myapp");

// Check if registered
if (PluginManager.has("app_myapp")) {
  // Plugin is available
}

// Get all plugins
const allPlugins = PluginManager.getAll();

// Get all app IDs
const appIds = PluginManager.getAppIds();
// → ["app_whatsapp", "app_phone", "app_myapp"]

// Get metadata for display
const meta = PluginManager.getMetadata("app_myapp");
// → { name: "My App", icon: "🚀", color: "#FF5722" }

// Get sound path
const sound = PluginManager.getSound("app_myapp", "notification");
// → "myapp-notification.mp3"
```

## Widget System

Widgets appear in platform-specific areas (Dynamic Island, status bar, lock screen).

### WidgetSlot Interface

```typescript
interface WidgetSlot {
  /** Where to render: dynamicIsland, statusBar, lockscreen, notification */
  mode: WidgetMode;

  /** Which platforms: ["ios"], ["android"], or ["ios", "android"] */
  platforms: Platform[];

  /** Priority when multiple widgets compete (higher wins) */
  priority: number;

  /** React component to render */
  component: WidgetComponent;

  /** Optional: expansion modes this widget supports */
  expansionModes?: ("minimal" | "compact" | "expanded")[];
}
```

### WidgetProps

```typescript
interface WidgetProps {
  /** App-specific state from world.appState[appId] */
  appState: any;

  /** Background app state (e.g., playing track info) */
  backgroundApp?: BackgroundAppState;

  /** Device profile for dimensions */
  deviceProfile: {
    dynamicIsland?: DynamicIslandConfig;
    statusBarWidget?: StatusBarWidgetConfig;
  };

  /** Current frame */
  currentFrame: number;

  /** Widget expansion mode */
  expansionMode: "minimal" | "compact" | "expanded";

  /** Platform */
  platform: Platform;
}
```

## Helper Functions

### definePlugin()

Create a plugin with sensible defaults:

```typescript
import { definePlugin } from "@tokovo/core";

const MyPlugin = definePlugin({
  id: "app_myapp",
  name: "My App",
  // version defaults to "1.0.0"
  // eventTypes defaults to []
  // sounds defaults to {}
});
```

### registerPlugins()

Register multiple plugins at once:

```typescript
import { registerPlugins } from "@tokovo/core";

registerPlugins([
  WhatsAppPlugin,
  PhonePlugin,
  TwitterPlugin,
  MyAppPlugin,
]);
```

## Debugging Plugins

| Issue | Cause | Fix |
|-------|-------|-----|
| App not rendering | Plugin not registered | Import in video-runner plugins.ts |
| Events not handled | Reducer not working | Check event.kind, event.appId |
| Camera not following | Behavior not registered | Call `BehaviorRegistry.register()` |
| Anchors not resolving | Provider not registered | Call `AnchorRegistry.register()` |
| Widgets not showing | Wrong platform/priority | Check `platforms` array and `priority` |
