## Debug Checklist

Before diving deep, check these common issues:

| Symptom | Quick Check |
|---------|-------------|
| Black screen | Is `foregroundAppId` set? |
| No messages | Is `conversationId` consistent? |
| Camera not moving | Is `directorEnabled={true}`? |
| No audio | Are sounds in `public/sounds/`? |
| "Module not found" | Did you run `pnpm build`? |

## Debugging Camera

<Tabs items={['No Movement', 'Wrong Target', 'Jittery']}>
  <Tab>
    **Problem:** Camera doesn't move at all

    **Causes:** `directorEnabled` is `false` or missing, no camera events in timeline, or anchor not found (falls back to device).

    **Debug:**

    ```tsx
    <TokovoRenderer
      world={world}
      t={frame}
      directorEnabled={true}
      directorDebug={true}
    />
    ```

    Check console for output like: `[Camera] Frame 30: Anchor "lastMessage" ŌåÆ rect {...}`
  </Tab>
  <Tab>
    **Problem:** Camera focuses on wrong element

    **Causes:** Anchor name mismatch (case-sensitive!), layout not computed yet, or element not visible at that frame.

    **Debug:**

    ```typescript
    console.log("Anchors:", AnchorRegistry.getAnchorsForApp("app_whatsapp"));
    console.log("Messages:", world.conversations["conv_1"]?.messages);
    ```

    **Common anchor names:** `lastMessage` (not `LastMessage`), `inputArea` (not `input_area`), `typingIndicator` (not `typing`).
  </Tab>
  <Tab>
    **Problem:** Camera is jittery/shaking too much

    **Causes:** Smoothing value too high, too many camera events in quick succession, or anchor position changing rapidly.

    **Fix:**

    ```typescript
    // Use lower smoothing for smoother motion
    dsl.camera.anchorTrack(60, "lastMessage", 40, 0.12);

    // Add holds between camera events
    dsl.camera.anchorFocus(60, "lastMessage", "message");
    dsl.camera.hold(85, 20);
    dsl.camera.anchorFocus(105, "lastMessage", "snap");
    ```
  </Tab>
</Tabs>

## Debugging State

### Inspect WorldState

```typescript
import { replay } from "@tokovo/core";

const world = replay(initialWorld, events, 60);

// Pretty print entire state
console.log(JSON.stringify(world, null, 2));

// Check specific parts
console.log("Device:", world.devices.phone);
console.log("Conversations:", world.conversations);
console.log("Camera:", world.camera);
```

### Trace Events

```typescript
// Add this to see every event as it's processed
events.forEach((e, i) => {
  if (e.at <= currentFrame) {
    console.log(`Event ${i}:`, e);
  }
});
```

### Frame-by-Frame Debugging

In Remotion Studio:
1. Open the timeline at bottom
2. Click on any frame
3. Add `console.log(world)` in your component
4. Check console for that frame's state

## Debugging Audio

| Symptom | Cause | Fix |
|---------|-------|-----|
| No sound | Wrong path or bus muted | Check file exists in `public/sounds/` |
| Too quiet | Ducking active | Higher-priority bus is playing |
| Cuts off early | Duration not set | Add `durationFrames` to sound event |
| Delayed | Wrong `at` frame | Verify frame number |

**Debug audio state:**
```typescript
console.log("Audio:", world.audio.activeSounds);
console.log("Buses:", world.audio.buses);
```

## Debugging Messages

### Messages Not Appearing

```typescript
// 1. Check the event is firing
console.log("Events at frame:", events.filter(e => e.at <= frame));

// 2. Check the conversation exists
console.log("Conversations:", Object.keys(world.conversations));

// 3. Check messages in conversation
console.log("Messages:", world.conversations["conv_1"]?.messages);

// 4. Verify conversation ID matches
// ŌØī Wrong
dsl.messages.receive(0, "conv1", "Alice", "Hey");  // no underscore
// Ō£ģ Correct
dsl.messages.receive(0, "conv_1", "Alice", "Hey"); // matches initial state
```

### Messages in Wrong Order

Check `at` frame values are sequential:
```typescript
const events = [
  dsl.messages.receive(0, "conv_1", "Alice", "First"),    // Frame 0
  dsl.messages.receive(30, "conv_1", "Bob", "Second"),    // Frame 30
  dsl.messages.receive(20, "conv_1", "Alice", "Third"),   // Frame 20 ŌåÉ OUT OF ORDER!
];
```

## Debugging Renderer

### Component Not Rendering

```typescript
// Check if app is registered
import { PluginManager } from "@tokovo/core";

console.log("Registered apps:", PluginManager.getAppIds());
console.log("WhatsApp:", PluginManager.getPlugin("app_whatsapp"));
```

### Wrong Device Profile

```typescript
// Check device profile
console.log("Profile:", world.devices.phone.profileId);

// Available profiles
import { DeviceProfileRegistry } from "@tokovo/devices";
console.log("Profiles:", DeviceProfileRegistry.list());
```

## Common Error Messages

| Error | Meaning | Fix |
|-------|---------|-----|
| `Cannot read property 'messages' of undefined` | Conversation doesn't exist | Create conversation in `initialWorld` |
| `Unknown app: app_xyz` | App not registered | Import the app package |
| `Anchor not found: xyz` | Invalid anchor name | Check spelling, use fallback |
| `Event at negative frame` | Frame < 0 | Fix event timing |

## Development Tools

### Remotion Studio Inspector

1. Open component tree (right panel)
2. Click any element
3. See its props and state

### React DevTools

1. Install React DevTools extension
2. Open browser DevTools ŌåÆ Components tab
3. Search for "TokovoRenderer"
4. Inspect all props

### Console Logging

```typescript
// In your video component
const frame = useCurrentFrame();
useEffect(() => {
  console.log(`Frame ${frame}:`, {
    world,
    events: events.filter(e => e.at === frame),
  });
}, [frame]);
```

## Getting Help

If you're stuck:

1. **Minimal reproduction** ŌĆö Simplest possible case
2. **Console output** ŌĆö Include relevant logs
3. **Frame number** ŌĆö Where does it break?
4. **Expected vs Actual** ŌĆö What should happen?
