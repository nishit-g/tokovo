## Setup Issues

### `pnpm install` fails

```bash
# Clear cache and reinstall
rm -rf node_modules
rm pnpm-lock.yaml
pnpm install
```

### `pnpm build` fails with type errors

```bash
# Rebuild all packages in order
pnpm build --filter @tokovo/ir
pnpm build --filter @tokovo/core
pnpm build --filter @tokovo/dsl
pnpm build --filter @tokovo/compiler
pnpm build --filter @tokovo/renderer
pnpm build
```

### "Module not found: @tokovo/xyz"

```bash
# Ensure all packages are built
pnpm build

# If still failing, check package is listed in dependencies
cat packages/your-package/package.json | grep "@tokovo"
```

## Runtime Errors

### "Cannot read property 'X' of undefined"

**In WorldState:**
```typescript
// ❌ Problem: Accessing undefined device
world.devices["phone"].foregroundAppId

// ✅ Solution: Check existence first
const device = world.devices["phone"];
if (device) {
  console.log(device.foregroundAppId);
}
```

**In conversations:**
```typescript
// ❌ Problem: Conversation not created
world.conversations["dm_bob"].messages

// ✅ Solution: Initialize in initialWorld
const initialWorld = {
  conversations: {
    dm_bob: {  // ← Must exist
      id: "dm_bob",
      name: "Bob",
      messages: [],
    },
  },
  // ...
};
```

### "Maximum call stack exceeded"

**Cause:** Circular reference in state or infinite reducer loop

**Fix:**
```typescript
// Check for self-referencing objects
// Use JSON.stringify to detect cycles
try {
  JSON.stringify(world);
} catch (e) {
  console.error("Circular reference in world state!");
}
```

## Rendering Issues

### Black/Empty Screen

| Check | Fix |
|-------|-----|
| `foregroundAppId` not set | Set in device state |
| App not imported | `import "@tokovo/apps-whatsapp"` |
| Wrong device ID | Match `focusDeviceId` prop |
| Build not run | `pnpm build` |

### White Screen with Error

Open browser DevTools Console. Common causes:
- Import error (missing package)
- Type error in component
- Invalid JSX

### UI Looks Wrong

| Issue | Check |
|-------|-------|
| Wrong colors | `platform` prop (ios/android) |
| Wrong layout | `profileId` in device state |
| Text cut off | Message `text` too long |

## Camera Issues

### Camera Doesn't Move

```tsx
// Check these:
<TokovoRenderer
  directorEnabled={true}   // ← Required
  focusDeviceId="phone"    // ← Must match device
  world={world}
  t={frame}
/>
```

### Camera Focuses Wrong Element

```typescript
// Anchor names are case-sensitive!
// ❌ Wrong
dsl.camera.anchorFocus(0, "LastMessage", "message");
dsl.camera.anchorFocus(0, "last_message", "message");

// ✅ Correct
dsl.camera.anchorFocus(0, "lastMessage", "message");
```

### Camera Jumps/Jitters

```typescript
// Add holds between camera events
dsl.camera.anchorFocus(0, "lastMessage", "message");
dsl.camera.hold(25, 15);  // ← Settle time
dsl.camera.anchorFocus(40, "lastMessage", "snap");
```

## Audio Issues

### No Sound

1. Check file exists: `public/sounds/notification.mp3`
2. Check path is correct: `/sounds/notification.mp3`
3. Check bus is not muted

### Sound Delayed

Frame numbers are exact. At 30fps:
- Frame 30 = 1 second
- Frame 60 = 2 seconds

### Sound Too Quiet

Higher-priority buses automatically duck lower ones:
- `ui` ducks everything
- `sfx` ducks `music`
- `voice` ducks `music`

## Remotion Issues

### Composition Not Listed

Ensure you added it to `Root.tsx`:
```tsx
<Composition
  id="MyVideo"           // ← Must be unique
  component={MyVideo}
  durationInFrames={180}
  fps={30}
  width={1080}
  height={1920}
/>
```

### Render Fails

```bash
# Check for errors in preview first
pnpm --filter video-runner dev

# Then render
pnpm --filter video-runner render MyVideo
```

### Export Creates Corrupt File

```bash
# Specify codec explicitly
pnpm --filter video-runner render MyVideo --codec h264
```

## Type Errors

### "Property X does not exist on type Y"

```typescript
// WorldState is strictly typed
// Cast if needed for custom properties
const state = world.appState.app_custom as CustomAppState;
```

### "Type 'string' is not assignable to type..."

```typescript
// Use const assertion for enums
const bus = "sfx" as const;

// Or use the enum
import { AudioBus } from "@tokovo/core";
const bus: AudioBus = "sfx";
```

## Performance Issues

### Slow Preview

1. Reduce video resolution temporarily
2. Close other browser tabs
3. Check for console logging in loop

### Render Takes Forever

```bash
# Use hardware acceleration
pnpm render --gl=angle

# Or reduce quality for testing
pnpm render --quality=50
```
