## How It Works

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   WRITE DSL     │ ──► │    COMPILE      │ ──► │     RENDER      │ ──► │      VIDEO      │
│  (Your Story)   │     │  (Scene→Frame)  │     │ (3-Engine Loop) │     │   (MP4/WebM)    │
└─────────────────┘     └─────────────────┘     └─────────────────┘     └─────────────────┘
       │                       │                       │
       ▼                       ▼                       ▼
   Author DSL              Compiler              Layout Engine
   or Events             Passes (5)            Camera Engine
                                                Audio Engine
```

## Quick Start

<Steps>

### Install

```bash
git clone https://github.com/your-org/tokovo.git && cd tokovo
pnpm install && pnpm build
```

### Run

```bash
pnpm --filter video-runner dev
# Open http://localhost:3000
```

### Create

```typescript
import { dsl } from "@tokovo/dsl";

const events = [
  dsl.messages.receive(0, "conv_1", "Alice", "Hey!"),
  dsl.camera.anchorFocus(0, "lastMessage", "message"),
  dsl.messages.send(60, "conv_1", "Hi Alice!"),
  dsl.camera.anchorFocus(60, "lastMessage", "snap"),
];
```

</Steps>

## Why Tokovo?

| Feature | Traditional | Tokovo |
|---------|-------------|--------|
| **Determinism** | Different renders each time | Same script = same video |
| **AI Generation** | Magic prompts, random output | Structured IR, predictable |
| **Scalability** | Manual editing per video | Batch-generate thousands |
| **Debugging** | "Something looks wrong" | Trace any frame to source |
| **Camera** | Manual keyframes | Semantic anchors ("lastMessage") |
| **Audio** | Manual sync | Bus-based mixer with ducking |

---

## The Four Layers

| Layer | Package | What It Does |
|-------|---------|-------------|
| **DSL** | `@tokovo/dsl` | Human-friendly story writing |
| **Scene IR** | `@tokovo/ir` | Semantic truth (no frames) |
| **Timeline IR** | `@tokovo/compiler` | Frame-based execution |
| **Runtime** | `@tokovo/core` + `@tokovo/renderer` | Renders pixels |

## Core Concepts

<Tabs items={['Episode', 'Event', 'WorldState', 'Camera']}>
  <Tab>
    **Episode**: A complete story containing devices, conversations, and beats.
    
    ```typescript
    const scene = episode("breakup-01", ep => {
      ep.device("AlicePhone", "iphone16", d => {
        d.conversation("dm_bob", { name: "Bob" });
        d.beat("tension", b => {
          b.receive("Bob", "We need to talk.");
        });
      });
    });
    ```
  </Tab>
  <Tab>
    **Event**: A frame-based instruction that mutates WorldState.
    
    ```typescript
    {
      at: 60,               // Frame number
      kind: "APP",          // Event category
      appId: "app_whatsapp",
      type: "MESSAGE_RECEIVED",
      text: "Hello!",
    }
    ```
  </Tab>
  <Tab>
    **WorldState**: The complete application state at any frame.
    
    ```typescript
    {
      devices: { phone: { isLocked: false, ... } },
      conversations: { dm_1: { messages: [...] } },
      appState: { app_whatsapp: { screen: "chat" } },
      camera: { transform: { scale: 1.08, ... } },
      audio: { activeSounds: { ... } },
    }
    ```
  </Tab>
  <Tab>
    **Camera**: Semantic anchors + shot presets for cinematic effects.
    
    ```typescript
    // Focus on the last message with dramatic zoom
    dsl.camera.anchorFocus(60, "lastMessage", "dramatic", 5);
    
    // Smooth tracking follow (webseries style)
    dsl.camera.anchorTrack(100, "inputArea", 40, 0.18);
    ```
  </Tab>
</Tabs>
