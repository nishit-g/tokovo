# Tokovo

Tokovo is a programmable phone-simulation experience for creators making shorts, reels, episodes, and series where the phone UI is part of the story.

It combines app plugins, deterministic runtime state, cinematic camera direction, and Remotion rendering so authored scenes can become repeatable phone-native videos.

## Architecture

- [`@tokovo/episodes`](/packages/episodes) owns the canonical episode surface.
- [`@tokovo/dsl`](/packages/dsl) and app plugins provide the authoring/runtime primitives.
- [`web`](/packages/web) is the public marketing shell.
- `video-runner` renders the final artifact.

## Flow

```text
Episode code
  -> compiler
  -> renderer
  -> video-runner
```

## New Episode Flow

1. Write the cinematic `*.episode.ts` file directly in `packages/episodes`.
2. Register the episode.
3. Preview through `video-runner`.
4. Render through `video-runner` or `@tokovo/render-service`.
