# Tokovo Documentation

Tokovo is a programmable phone-simulation engine for creators making shorts, reels, episodes, and series where the phone UI is part of the story.

It combines deterministic app state, device simulation, cinematic camera direction, audio, and Remotion rendering so a scripted scene can become a repeatable portrait video.

## Core Ideas

- Episodes are code: timing, app seed data, camera, audio, and render settings live in `packages/episodes`.
- The runtime is deterministic: the same episode and frame should resolve the same state.
- App packages are plugins: each simulator owns its reducer, views, anchors, layouts, and DSL helpers.
- `video-runner` is the local Remotion surface for preview and MP4 export.
- `@tokovo/render-service` provides the render-service CLI and orchestration primitives.

## Current V1 Surface

Tokovo includes app simulators for WhatsApp, iMessage, Instagram, X, LinkedIn, Snapchat, Teams, and Typewriter. It also includes device-owned systems for camera direction, keyboard input, notifications, overlays, backgrounds, audio, and voice tracks.

## Release Confidence

The repository has one canonical release gate:

```bash
pnpm verify:release
```

During feature work, use narrower checks first:

```bash
pnpm -s typecheck:solution
pnpm validate
pnpm --filter @tokovo/episodes test
pnpm --filter video-runner typecheck
```
