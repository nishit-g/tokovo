## Camera as Storytelling Tool

In traditional filmmaking, camera work guides the audience's emotional journey. Tokovo brings this power to phone simulations:

- **Focus** directs attention to specific UI elements (messages, typing indicators, notifications)
- **Zoom** creates emphasis or context (zoom in for intimacy, zoom out for overview)
- **Shake** adds visceral impact (shock, urgency, emotion)

Unlike static screen recordings, camera choreography creates **cinematic rhythm** that matches the emotional beats of your story.

## Camera Fundamentals

### Focus System

The `focus()` method directs the camera to frame specific UI elements using semantic anchors.

```typescript
.camera((cam) => {
  cam.at('0s').focus('deviceScreen')    // Wide establishing shot
  cam.at('2s').focus('lastMessage')     // Close-up on newest message
  cam.at('5s').focus('notification')    // Shift to notification banner
})
```

**Common Focus Anchors:**

- `deviceScreen` — Full device view (establishing shot)
- `lastMessage` — Most recent message in chat
- `typingIndicator` — Typing bubble animation
- `chatHeader` — Contact name/avatar area
- `messageInput` — Text input field
- `notification` — System notification banner

Focus transitions are smooth by default. The camera eases between targets, creating professional motion.

### Zoom Control

The `zoom()` method adjusts the camera's magnification level.

```typescript
cam.at("0s").zoom(1.0); // Normal framing
cam.at("3s").zoom(1.5); // Zoom in 50% (emphasis, intimacy)
cam.at("6s").zoom(0.8); // Zoom out 20% (context, overview)
```

**Zoom Guidelines:**

- **1.0** — Default, balanced framing
- **0.8-0.9** — Wide shot (establishing, context)
- **1.2-1.5** — Medium close-up (emphasis, focus)
- **1.6-2.0** — Extreme close-up (dramatic reveals, shock)

Combine zoom with focus for powerful effects:

```typescript
// Start wide, then push in for emphasis
cam.at("0s").zoom(0.8);
cam.at("0s").focus("deviceScreen");
cam.at("2s").zoom(1.3);
cam.at("2s").focus("lastMessage");
```

### Camera Shake

The `shake()` method adds dynamic motion to simulate impact, emotion, or urgency.

```typescript
cam.at("8s").shake(0.5); // Subtle shake (notification arrives)
cam.at("10s").shake(1.0); // Medium shake (shock, surprise)
cam.at("12s").shake(2.0); // Intense shake (dramatic impact)
```

**Shake Intensity Scale:**

- **0.3-0.5** — Subtle (notifications, typing nervously)
- **0.8-1.2** — Noticeable (emotional reactions, unexpected messages)
- **1.5-2.5** — Intense (rage, devastation, plot twists)

Shake duration is brief (typically 0.3-0.5 seconds). Use sparingly for maximum impact.

## Cinematic Patterns

These battle-tested patterns create professional camera work for common storytelling scenarios.

### Pattern 1: Opening Establishing Shot

Start with context, then guide the viewer into the story.

```typescript
.camera((cam) => {
  // Begin wide to show full device
  cam.at('0s').zoom(0.8)
  cam.at('0s').focus('deviceScreen')

  // Gradually move to normal view
  cam.at('1.5s').zoom(1.0)

  // Then focus on the action
  cam.at('2s').focus('lastMessage')
})
```

**Why it works:** Establishes location and context before diving into detail. Mirrors how films open with wide shots before cutting to close-ups.

### Pattern 2: Message Reveal (Building Tension)

Build anticipation before delivering the key message.

```typescript
.camera((cam) => {
  // Start on header (who's messaging?)
  cam.at('0s').focus('chatHeader')

  // Show typing indicator (what will they say?)
  cam.at('1s').focus('typingIndicator')

  // The reveal — message appears
  cam.at('3s').focus('lastMessage')

  // Emphasize with zoom
  cam.at('3.5s').zoom(1.3)
})
```

**Why it works:** Creates narrative suspense. The viewer wonders what's being typed, then the camera emphasizes the payoff.

### Pattern 3: Shock Reaction

Immediate, visceral response to surprising or devastating news.

```typescript
.camera((cam) => {
  // Impact moment
  cam.at('5s').shake(1.5)

  // Quick zoom to emphasize shock
  cam.at('5.2s').zoom(1.4)

  // Hold focus on the message
  cam.at('5s').focus('lastMessage')
})
```

**Why it works:** Shake communicates emotional response, zoom locks viewer attention on the critical message.

### Pattern 4: Multi-Message Flow

Smoothly follow a conversation exchange.

```typescript
.camera((cam) => {
  // Each message gets focused attention
  cam.at('1s').focus('lastMessage')   // First message
  cam.at('4s').focus('lastMessage')   // Reply
  cam.at('7s').focus('lastMessage')   // Next message
  cam.at('10s').focus('lastMessage')  // Continue...

  // Zoom in during emotional peak
  cam.at('7s').zoom(1.2)
})
```

**Why it works:** `lastMessage` anchor automatically updates to the newest message, creating smooth tracking.

## Advanced Techniques

### Smooth Transitions

Camera movements should **match message pacing**. Avoid jarring cuts.

```typescript
// ❌ Bad — instant jumps feel unnatural
cam.at("2s").zoom(1.0);
cam.at("2s").zoom(1.5); // Same timestamp

// ✅ Good — gradual transitions
cam.at("2s").zoom(1.0);
cam.at("3.5s").zoom(1.5); // Smooth 1.5s transition
```

**Timing Guidelines:**

- Focus changes: 0.5-1.5s between shifts
- Zoom changes: 1.0-2.0s for smooth motion
- Shake: Instant (shake is brief by design)

### Anchor-Based Framing

Apps register **semantic anchors** for dynamic UI elements. You can also register custom anchors for precise framing.

```typescript
// Inside an app plugin
registerAnchor("reactionArea", {
  x: 0.5, // Centered horizontally
  y: 0.7, // 70% down screen
  w: 0.8, // 80% of screen width
  h: 0.2, // 20% of screen height
});

// Use in camera choreography
cam.at("3s").focus("reactionArea");
```

**Common Custom Anchors:**

- Specific message by ID (`message:abc123`)
- Reply chain region
- Profile picture area
- Action buttons (send, attach, etc.)

### Multi-Device Compositions

For episodes with multiple devices, prefix anchors with device IDs.

```typescript
.camera((cam) => {
  // Focus on sender's device
  cam.at('0s').focus('device1:lastMessage')

  // Cut to receiver's device
  cam.at('2s').focus('device2:lastMessage')

  // Wide shot showing both devices
  cam.at('4s').focus('deviceScreen')
  cam.at('4s').zoom(0.6)
})
```

This technique enables split-perspective storytelling (sender vs. receiver viewpoints).

## Complete Example: Breakup Scene

A 30-second cinematic sequence combining all techniques.

```typescript
episode("breakup", { duration: 30 })
  .device("phone", "iphone16", (d) => {
    d.app("app_whatsapp", (wa) => {
      wa.at("1s").receive("Sarah", "We need to talk...");
      wa.at("5s").typing("Sarah");
      wa.at("8s").receive("Sarah", "I think we should break up.");
      wa.at("12s").typing("You");
      wa.at("15s").send("You", "What? Why?");
      wa.at("18s").typing("Sarah");
      wa.at("22s").receive("Sarah", "I'm sorry. It's not working.");
      wa.at("26s").typing("You");
    });
  })
  .camera((cam) => {
    // Opening: Wide, ominous
    cam.at("0s").zoom(0.9);
    cam.at("0s").focus("deviceScreen");

    // First message arrives — "We need to talk"
    cam.at("1s").focus("lastMessage");
    cam.at("1.5s").zoom(1.1); // Slight emphasis

    // Tension builds — Sarah typing
    cam.at("5s").focus("typingIndicator");

    // The bombshell — "I think we should break up"
    cam.at("8s").focus("lastMessage");
    cam.at("8.1s").shake(1.2); // Emotional impact
    cam.at("8.5s").zoom(1.4); // Lock attention

    // User reacts, starts typing desperately
    cam.at("12s").zoom(1.0); // Pull back slightly
    cam.at("12s").focus("messageInput");

    // Send desperate response — "What? Why?"
    cam.at("15s").focus("lastMessage");
    cam.at("15s").shake(0.5); // Shaky hands

    // Sarah typing again
    cam.at("18s").focus("typingIndicator");

    // Final blow — "I'm sorry. It's not working."
    cam.at("22s").focus("lastMessage");
    cam.at("22.2s").shake(1.8); // Devastation
    cam.at("22.5s").zoom(1.5); // Intimate pain

    // User starts typing, but camera holds on Sarah's message
    cam.at("26s").zoom(1.6); // Slow push, lingering
  });
```

**Scene Breakdown:**

1. **Setup (0-5s):** Wide shot establishes tone. First message creates unease.
2. **Rising tension (5-8s):** Focus on typing builds suspense.
3. **Climax (8-15s):** Shake + zoom = visceral shock. Camera mimics emotional reaction.
4. **Response (15-22s):** Desperate typing, shaky camera = panic.
5. **Resolution (22-30s):** Final shake and slow zoom = devastation, finality.

**Camera Storytelling Principles:**

- Zoom mirrors emotional intensity (wide = calm, tight = intense)
- Shake communicates visceral reactions
- Focus guides narrative attention
- Timing matches emotional beats

## Best Practices

### 1. Start Wide, End Tight (or Vice Versa)

Open with context, close on emotion:

```typescript
cam.at("0s").zoom(0.8); // Establish
// ... story unfolds
cam.at("25s").zoom(1.5); // Emotional close
```

Or reverse for flashbacks/reveals:

```typescript
cam.at("0s").zoom(1.5); // Mysterious close-up
// ... context revealed
cam.at("25s").zoom(0.8); // Pull back to show bigger picture
```

### 2. Match Camera to Emotion

- **Joy, excitement:** Quick focus shifts, moderate zoom
- **Sadness, loss:** Slow movements, gradual zooms
- **Anger, conflict:** Sharp cuts, intense shake
- **Suspense:** Linger on typing, delayed focus changes

### 3. Don't Overuse Shake

Shake is powerful but loses impact with overuse. Reserve for:

- Major plot twists
- Emotional outbursts
- Shocking revelations

**Rule of thumb:** 2-3 shakes per 30-second scene maximum.

### 4. Time Transitions to Content

Align camera movements with message delivery:

```typescript
// ✅ Good — camera moves match action
wa.at("3s").receive("Alice", "I love you");
cam.at("3s").focus("lastMessage");
cam.at("3.2s").zoom(1.3);

// ❌ Bad — camera moves randomly
wa.at("3s").receive("Alice", "I love you");
cam.at("1s").focus("lastMessage"); // Too early
cam.at("5s").zoom(1.3); // Too late
```

### 5. Use Focus to Guide Viewer Attention

The viewer can only look at one thing at a time. Use focus deliberately:

```typescript
// Tell the story through camera focus
cam.at("0s").focus("chatHeader"); // Who?
cam.at("1s").focus("typingIndicator"); // Anticipation
cam.at("3s").focus("lastMessage"); // The reveal
```

### 6. Establish, Then Explore

Begin scenes with wide shots, then move closer:

```typescript
cam.at("0s").focus("deviceScreen"); // Full context
cam.at("2s").focus("lastMessage"); // Dive into detail
```

This mirrors cinematic language audiences instinctively understand.

## Common Mistakes

### ❌ Too Many Rapid Movements

```typescript
// ❌ Bad — viewer whiplash
cam.at("1s").focus("chatHeader");
cam.at("1.2s").focus("lastMessage");
cam.at("1.4s").focus("typingIndicator");
cam.at("1.6s").zoom(1.5);
```

**Fix:** Space out movements. Let each shot breathe.

```typescript
// ✅ Good — deliberate pacing
cam.at("1s").focus("chatHeader");
cam.at("3s").focus("lastMessage");
cam.at("6s").focus("typingIndicator");
```

### ❌ Unmotivated Zooms

```typescript
// ❌ Bad — why are we zooming here?
wa.at("3s").receive("Alice", "What's for dinner?");
cam.at("3s").zoom(1.8); // Extreme zoom on mundane message
```

**Fix:** Zoom should match emotional weight.

```typescript
// ✅ Good — zoom matches importance
wa.at("3s").receive("Alice", "I'm pregnant.");
cam.at("3s").zoom(1.8); // Zoom justified by revelation
```

### ❌ Ignoring Message Pacing

```typescript
// ❌ Bad — camera doesn't follow the conversation
wa.at("2s").receive("Bob", "Hello");
wa.at("4s").receive("Bob", "Are you there?");
wa.at("6s").receive("Bob", "Please respond");
// No camera movement
```

**Fix:** Track the conversation flow.

```typescript
// ✅ Good — camera follows messages
wa.at("2s").receive("Bob", "Hello");
cam.at("2s").focus("lastMessage");

wa.at("4s").receive("Bob", "Are you there?");
cam.at("4s").focus("lastMessage");
cam.at("4.5s").zoom(1.2); // Tension building

wa.at("6s").receive("Bob", "Please respond");
cam.at("6s").focus("lastMessage");
cam.at("6.1s").shake(0.8); // Desperation
```

### ❌ Forgetting Establishing Shots

```typescript
// ❌ Bad — viewer is disoriented from start
cam.at("0s").zoom(1.5);
cam.at("0s").focus("lastMessage"); // What app? What device?
```

**Fix:** Establish context first.

```typescript
// ✅ Good — orient the viewer
cam.at("0s").zoom(0.8);
cam.at("0s").focus("deviceScreen"); // Show full context
cam.at("2s").zoom(1.0);
cam.at("2s").focus("lastMessage"); // Then dive in
```
