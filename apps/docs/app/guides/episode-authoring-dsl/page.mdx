# Episode Authoring DSL

Tokovo episode code stays code-first.

Use the DSL for:

- device bootstrap
- app snapshots and explicit initial views
- timing
- camera
- audio
- per-app sequencing

Typical shape:

```ts
const ep = episode("demo", { fps: 30, duration: "10s", title: "Demo" })
  .device("phone", "iphone16", { app: "app_whatsapp" })
  .snapshot("app_whatsapp", "phone", {
    conversations: [{ id: "dm_alex", name: "Alex", messages: [] }],
  })
  .view("app_whatsapp", "phone", {
    screen: "chat",
    conversationId: "dm_alex",
  });
```

`snapshot()` and `view()` are the only supported frame-0 app bootstrap APIs. Do not put app history into `device()` and do not rely on compiler-owned defaults.
