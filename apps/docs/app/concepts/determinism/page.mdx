# Determinism

Determinism is the core Tokovo invariant.

## Rule

Same authored input, same manifest artifacts, same runtime registrations, and same frame should produce the same render output.

## What Supports That

- `@tokovo/core` replay stays headless
- renders do not depend on live Payload reads
- runtime plugin registration is explicit
- manifest artifacts are committed into the repo

Without this property, reproducible rendering and trustworthy debugging break down.
