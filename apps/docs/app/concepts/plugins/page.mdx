# Plugin System

Tokovo uses registries and plugins so app-specific and device-specific behavior stays out of `@tokovo/core`.

## What A Plugin Owns

A plugin package usually owns some combination of:

- runtime reducers
- track builders and DSL helpers
- React views
- anchors or layout helpers
- package-specific UI rules

Examples:

- `@tokovo/apps-whatsapp`
- `@tokovo/apps-x`
- `@tokovo/device-camera`
- `@tokovo/device-notifications`
- `@tokovo/overlay`

## Registration Model

Plugins are registered by the runtime app, not by import-time side effects.

`video-runner` does this explicitly:

- register app plugins
- register device plugins
- register notifications, camera, keyboard, overlay
- register anchor providers

That keeps the runtime boot path inspectable and testable.

## Why This Matters

- `core` remains deterministic and headless
- `renderer` can render through registries rather than app-specific imports
- new app packages can be added without rewriting the engine
