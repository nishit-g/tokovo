# Anchors

Anchors are semantic camera and layout targets in Tokovo.

## Why They Exist

Episodes should focus on meaningful UI regions, not brittle pixel coordinates.

Examples:

- `device`
- `app`
- `lastMessage`
- `typingIndicator`
- app-specific thread or composer regions

## Role In The System

- app packages expose anchor-aware layout regions
- camera and renderer code resolve those regions at runtime
- episodes target anchors for focus and tracking

Use stable semantic anchors whenever possible.
