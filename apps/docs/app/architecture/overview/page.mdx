# Overview

Tokovo separates authoring from choreography.

- Episodes are authored directly in code.
- App bootstrap is always `device + snapshot + view + timeline`.
- Runtime registration is explicit through checked-in catalogs.
- `*.episode.ts` files own scene timing, camera, and app sequencing.
- Renders stay deterministic because they consume committed code, not live CMS reads.

Tokovo does not support device-owned app history bootstrap. Previous app data must come from plugin-owned snapshots, and frame-0 UI must come from explicit initial views.
