# @tokovo/studio

> **Priority: P2** — Visual development tools for building and previewing Tokovo episodes.

---

## Overview

`@tokovo/studio` is a **Vite-based development application** providing visual tools for episode creation, timeline editing, and real-time preview. It bridges the gap between code-based episode authoring and visual feedback, making it easier to craft polished phone-simulation narratives.

Unlike `@tokovo/react` (which provides React primitives) or `@tokovo/renderer` (which renders episodes to video), Studio is a **standalone development application** built on top of these packages.

### Installation

Studio is a workspace package in the Tokovo monorepo:

```bash
# From monorepo root
pnpm install

# Start Studio development server
pnpm --filter @tokovo/studio dev
```

**Note:** `@tokovo/studio` is currently a private workspace package, not published to npm. It's designed for internal development workflows.

### Role in Architecture

Studio sits at the **developer interface layer**, consuming lower-level packages:

```
Episode JSON/TypeScript (DSL)
         ↓
   @tokovo/compiler → TrackEpisodeIR
         ↓
   @tokovo/core → WorldState
         ↓
   @tokovo/renderer → Visual output
         ↓
   @tokovo/studio → Interactive preview + editing
```

## What is Studio?

Studio is a **visual development environment** for Tokovo episodes, offering:

### 1. **Episode Preview**

Real-time playback of episodes using Remotion Player + TokovoRenderer

### 2. **Timeline Visualization** _(Planned)_

Visual timeline showing:

- Camera events (zoom, focus, shake)
- Audio events (BGM, sound effects)
- Device events (lock, unlock, app open)
- App events (messages, typing, notifications)

### 3. **Frame Scrubbing** _(Planned)_

Click-to-seek navigation through episode timeline

### 4. **Episode Inspector** _(Planned)_

Debug panel showing:

- Current WorldState
- Active events at playhead
- Device state inspection
- App state inspection

## When to Use Studio

Use `@tokovo/studio` when:

- ✅ **Previewing episodes** during development
- ✅ **Debugging episode timing** and event sequences
- ✅ **Visualizing complex timelines** with multiple event tracks
- ✅ **Experimenting** with camera movements and transitions
- ✅ **Quality-checking** before rendering to video

Use other packages when:

- ❌ **Rendering final videos** → Use `@tokovo/renderer` in `video-runner`
- ❌ **Building custom editors** → Use `@tokovo/react` primitives
- ❌ **Authoring episodes** → Use `@tokovo/dsl` or write IR JSON

## Current Features

### Episode Renderer

Studio provides an `EpisodeRenderer` component that wraps `TokovoRenderer` for preview contexts:

```typescript
import { EpisodeRenderer } from "@tokovo/studio";
import type { PreparedTrackEpisode } from "@tokovo/compiler";

function PreviewPanel({ episode }: { episode: PreparedTrackEpisode }) {
  const [frame, setFrame] = useState(0);

  return (
    <div className="preview-container">
      <EpisodeRenderer episodeIR={episode} frame={frame} />

      <input
        type="range"
        min={0}
        max={episode.durationInFrames}
        value={frame}
        onChange={(e) => setFrame(Number(e.target.value))}
      />
    </div>
  );
}
```

**Key Features:**

- Memoized world state computation via `runEpisode()`
- Preview mode rendering (optimized for interactive scrubbing)
- Loading state handling
- Dark theme background (`#0a0a0f`)

## Building Custom Editors

If Studio doesn't meet your needs, you can build custom editors using `@tokovo/react`:

```typescript
import { useState } from "react";
import { TokovoProvider, useDevice, useWorld } from "@tokovo/react";
import { runEpisode, createTokovoRegistries, PluginManagerClass } from "@tokovo/core";
import type { PreparedTrackEpisode } from "@tokovo/compiler";

function CustomStudio({ episode }: { episode: PreparedTrackEpisode }) {
  const [frame, setFrame] = useState(0);
  const world = runEpisode(episode, frame);

  return (
    <div className="studio-layout">
      {/* Preview Panel */}
      <div className="preview-panel">
        <TokovoProvider
          world={world}
          deviceId="phone"
          appId="app_whatsapp"
          t={frame}
        >
          <PhonePreview />
        </TokovoProvider>
      </div>

      {/* Timeline Panel (custom implementation) */}
      <div className="timeline-panel">
        <CustomTimeline
          events={episode.events}
          currentFrame={frame}
          onSeek={setFrame}
        />
      </div>

      {/* Inspector Panel */}
      <div className="inspector-panel">
        <WorldInspector world={world} />
        <EventList events={episode.events} currentFrame={frame} />
      </div>
    </div>
  );
}

function PhonePreview() {
  const device = useDevice();
  const world = useWorld();

  return (
    <div className="phone-container">
      <div className="device-info">
        Device: {device.id} |
        Locked: {device.isLocked ? "🔒" : "🔓"} |
        App: {device.foregroundAppId || "None"}
      </div>
      {/* Render device UI */}
    </div>
  );
}
```

**When to build custom:**

- You need **domain-specific editing features** not in Studio
- You want **custom timeline visualizations** (e.g., musical notation)
- You need **brand-specific UI/UX** for episode authoring
- You're building a **SaaS product** around Tokovo

## Example: Minimal Preview App

Here's a minimal episode preview using Studio's architecture:

```typescript
import { useState, useMemo } from "react";
import { runEpisode } from "@tokovo/core";
import { TokovoRenderer } from "@tokovo/renderer";
import type { PreparedTrackEpisode } from "@tokovo/compiler";

function MinimalStudio({ episode }: { episode: PreparedTrackEpisode }) {
  const [frame, setFrame] = useState(0);
  const [playing, setPlaying] = useState(false);
  const registries = useMemo(() => createTokovoRegistries(), []);
  const pluginManager = useMemo(
    () => new PluginManagerClass(registries.plugins),
    [registries],
  );

  const world = useMemo(
    () =>
      runEpisode(episode, frame, {
        mode: "preview",
        registries: registries.engine,
      }),
    [episode, frame, registries]
  );

  return (
    <div className="studio">
      <div className="preview">
        <TokovoRenderer
          world={world}
          t={frame}
          debug={false}
          pluginManager={pluginManager}
          registries={registries.plugins}
        />
      </div>

      <div className="controls">
        <button onClick={() => setPlaying(!playing)}>
          {playing ? "⏸️ Pause" : "▶️ Play"}
        </button>

        <input
          type="range"
          min={0}
          max={episode.durationInFrames}
          value={frame}
          onChange={(e) => setFrame(Number(e.target.value))}
        />

        <span>
          Frame {frame} / {episode.durationInFrames} ({episode.fps} fps)
        </span>
      </div>
    </div>
  );
}
```
