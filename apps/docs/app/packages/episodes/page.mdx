# @tokovo/episodes

Canonical Tokovo episode surface.

It owns:

- episode definitions
- catalog taxonomy and metadata
- curated app showcases, system showcases, and stories
- validation helpers

## Catalog Model

Tokovo now uses a storybook-style catalog model with canonical `catalogType` values:

- `app_showcase_flagship`
- `app_showcase_exhaustive`
- `app_showcase_theme`
- `system_showcase`
- `story`
- `test`

Catalog browsing and filtering should use `catalogType`.
