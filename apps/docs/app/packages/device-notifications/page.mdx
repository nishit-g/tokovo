# @tokovo/device-notifications

> Enterprise-grade notification system for Tokovo phone simulations.

---

## Overview

`@tokovo/device-notifications` is a **P3 priority plugin** that brings OS-level notification functionality to Tokovo episodes. Display push notifications, banner alerts, notification centers, badges, and Dynamic Island interactions with a fluent DSL.

**Priority**: P3 (optional device feature)  
**Category**: Device Plugin  
**Platforms**: iOS, Android

---

## Installation

```bash
pnpm add @tokovo/device-notifications
```

---

## Quick Start

```typescript
import { NotificationTrackBuilder } from "@tokovo/device-notifications";
import { episode } from "@tokovo/dsl";

let order = 0;
const getOrder = () => order++;

episode("story", { fps: 30, duration: "10s" })
  .device("phone", "iphone16", {
    app: "app_whatsapp",
  })
  .track(
    "notifications",
    () => {
      return new NotificationTrackBuilder(30, "phone", getOrder);
    },
    (noti) => {
      // Show heads-up notification
      noti.at("2s").show({
        id: "notif_1",
        appId: "app_whatsapp",
        title: "Alice",
        body: "Hey! Are you free tonight?",
        mode: "headsup",
      });

      // Tap to open app
      noti.at("5s").tap("notif_1");

      // Dismiss notification
      noti.at("8s").dismiss("notif_1");
    },
  )
  .build();
```

## Features

### Core Notification Types

| Type           | Description              | Behavior                                   |
| -------------- | ------------------------ | ------------------------------------------ |
| **Heads-Up**   | Banner at top of screen  | Auto-dismisses after 3-5s, can tap to open |
| **Lockscreen** | Shows on locked device   | Persists until dismissed or unlocked       |
| **Silent**     | No banner, goes to shade | Only visible in notification center        |
| **Badge**      | App icon badge only      | Number indicator, no banner                |

### Notification Behaviors

- **Push notifications** - Incoming message/alert display
- **Banner notifications** - Temporary top-of-screen alerts
- **Notification center** - Persistent notification tray
- **Notification badges** - App icon number indicators
- **Notification sounds** - Audio alerts (iOS/Android sounds)
- **Notification interactions** - Tap, swipe, dismiss actions
- **Dynamic Island** - iOS 16+ live activity integration

## Notification DSL

### NotificationTrackBuilder

The main entry point for authoring notification events.

```typescript
const noti = new NotificationTrackBuilder(fps, deviceId, getOrder);

noti.at("2s").show({ ... });
noti.at("5s").dismiss("notif_1");
noti.at("7s").tap("notif_2");
```

### Time Positioning

```typescript
// String time (recommended)
noti.at("2s").show({ ... });      // 2 seconds
noti.at("500ms").show({ ... });   // 500 milliseconds

// Frame number
noti.at(60).show({ ... });        // Frame 60

// Span (notifications use start time)
noti.span("2s", "5s").show({ ... }); // Shows at 2s
```

## Notification Methods

### .show(opts)

Display a notification.

```typescript
noti.at("2s").show({
  id: "notif_1", // Unique ID (required)
  appId: "app_whatsapp", // App that sent it (required)
  title: "Alice", // Notification title (required)
  body: "Hey there!", // Message text (required)
  mode: "headsup", // Display mode (optional)
  priority: "high", // Priority level (optional)
  icon: "/avatars/alice.jpg", // App icon (optional)
  preview: {
    // Rich preview (optional)
    kind: "image",
    value: "/previews/photo.jpg",
    aspectRatio: 16 / 9,
  },
  actions: [
    // Quick actions (optional)
    {
      id: "reply",
      label: "Reply",
      icon: "💬",
    },
    {
      id: "archive",
      label: "Archive",
      icon: "📦",
    },
  ],
  groupKey: "dm_alice", // Stack notifications (optional)
  threadId: "conv_123", // Conversation thread (optional)
  replyable: true, // Allow inline reply (optional)
});
```

### .dismiss(id)

Dismiss a notification.

```typescript
// Dismiss specific notification
noti.at("5s").dismiss("notif_1");

// Dismiss by options
noti.at("5s").dismiss({
  id: "notif_1",
});
```

### .tap(id, actionId?)

Tap a notification to open the app.

```typescript
// Simple tap (opens app)
noti.at("3s").tap("notif_1");

// Tap action button
noti.at("3s").tap("notif_1", "reply");
```

### .swipe(id, direction, action)

Swipe notification to perform action.

```typescript
// Swipe left to dismiss
noti.at("4s").swipe("notif_1", "left", "dismiss");

// Swipe right to archive
noti.at("4s").swipe("notif_1", "right", "archive");
```

**Available actions**: `"dismiss"`, `"archive"`, `"snooze"`, `"mark_read"`

### .dynamicIsland(opts)

Update Dynamic Island state (iOS 16+).

```typescript
noti.at("2s").dynamicIsland({
  mode: "compact", // "compact" | "expanded" | "minimal"
  appId: "app_music", // App controlling island
});
```

### .openPanel()

Open notification shade/panel.

```typescript
noti.at("5s").openPanel();
```

### .closePanel()

Close notification shade/panel.

```typescript
noti.at("8s").closePanel();
```

### .clearAll()

Clear all notifications.

```typescript
noti.at("10s").clearAll();
```

### .reply(id, text)

Send inline reply to notification.

```typescript
noti.at("6s").reply("notif_1", "Thanks!");
```

## Notification Properties

### Required Properties

| Property | Type     | Description                          |
| -------- | -------- | ------------------------------------ |
| `id`     | `string` | Unique notification identifier       |
| `appId`  | `string` | App that sent the notification       |
| `title`  | `string` | Notification title (sender/app name) |
| `body`   | `string` | Notification message text            |

### Optional Properties

| Property    | Type                   | Default     | Description                                      |
| ----------- | ---------------------- | ----------- | ------------------------------------------------ |
| `mode`      | `NotificationMode`     | `"headsup"` | Display mode (headsup, lockscreen, both, silent) |
| `priority`  | `NotificationPriority` | `"default"` | Priority level (low, default, high, urgent)      |
| `icon`      | `string`               | App default | App icon URL or emoji                            |
| `preview`   | `PreviewContent`       | -           | Rich media preview (text, image, video)          |
| `actions`   | `Action[]`             | -           | Quick action buttons                             |
| `groupKey`  | `string`               | -           | Group notifications together                     |
| `threadId`  | `string`               | -           | Conversation thread ID                           |
| `replyable` | `boolean`              | `false`     | Allow inline reply                               |
| `metadata`  | `Record<string, any>`  | -           | App-specific data                                |

### NotificationMode

```typescript
type NotificationMode =
```

### NotificationPriority

```typescript
type NotificationPriority =

  | "urgent"; // Full-screen alert
```

## Notification Lifecycle

### State Transitions

```
pending → headsUp → inShade → dismissed
   ↓                  ↓
lockscreen ──────────┘
```

**States**:

- **pending** - Scheduled, not yet shown
- **headsUp** - Banner visible at top of screen
- **inShade** - In notification center/tray
- **dismissed** - Removed from all views

### Timing Properties

Notifications track their lifecycle with frame-based timing:

| Property           | Type     | Description                           |
| ------------------ | -------- | ------------------------------------- |
| `createdAtFrame`   | `number` | Frame when notification was created   |
| `shownAtFrame`     | `number` | Frame when banner appeared            |
| `dismissedAtFrame` | `number` | Frame when notification was dismissed |

## Plugin Architecture

### Plugin Registration

The notifications plugin integrates with Tokovo's core plugin system:

```typescript
import { registerNotificationPlugin } from "@tokovo/device-notifications";

// Register plugin (automatic in video-runner)
registerNotificationPlugin();
```

**What gets registered**:

- **Reducer**: Handles all `NOTIFICATION_*` events
- **Audio Rules**: Auto-plays notification sounds
- **Sounds**: Default iOS/Android notification tones

### Event Types

The plugin handles these event types:

- `NOTIFICATION_SHOW` - Display notification
- `NOTIFICATION_DISMISS` - Remove notification
- `NOTIFICATION_TAP` - User tapped notification
- `NOTIFICATION_SWIPE` - User swiped notification
- `NOTIFICATION_DYNAMIC_ISLAND` - Update Dynamic Island
- `NOTIFICATION_OPEN_PANEL` - Open notification center
- `NOTIFICATION_CLOSE_PANEL` - Close notification center
- `NOTIFICATION_CLEAR_ALL` - Clear all notifications
- `NOTIFICATION_REPLY` - Inline reply sent

### State Management

Notifications are stored in the device state:

```typescript
interface DeviceState {
  notifications: {
    instances: NotificationInstance[]; // Active notifications
    panel: {
      isOpen: boolean;
      scrollOffset: number;
    };
    dynamicIsland: DynamicIslandState;
  };
}
```

## Complete Example

```typescript
import { episode } from "@tokovo/dsl";
import { NotificationTrackBuilder } from "@tokovo/device-notifications";
import { WhatsAppTrackBuilder } from "@tokovo/apps-whatsapp";

let order = 0;
const getOrder = () => order++;

episode("notification-story", { fps: 30, duration: "20s" })
  .device("phone", "iphone16", {
    app: "app_whatsapp",
    conversations: [
      {
        id: "dm_alice",
        name: "Alice 👋",
        avatar: "/avatars/alice.jpg",
      },
    ],
  })

  // Notification track
  .track(
    "notifications",
    () => {
      return new NotificationTrackBuilder(30, "phone", getOrder);
    },
    (noti) => {
      // First notification: Message from Alice
      noti.at("1s").show({
        id: "notif_1",
        appId: "app_whatsapp",
        title: "Alice 👋",
        body: "Hey! Are you free tonight?",
        mode: "headsup",
        priority: "high",
        icon: "/avatars/alice.jpg",
        preview: {
          kind: "text",
          value: "Hey! Are you free tonight?",
        },
        replyable: true,
      });

      // Second notification: Another message
      noti.at("4s").show({
        id: "notif_2",
        appId: "app_whatsapp",
        title: "Alice 👋",
        body: "Check out this photo! 📸",
        mode: "headsup",
        preview: {
          kind: "image",
          value: "/images/photo.jpg",
          aspectRatio: 16 / 9,
        },
      });

      // Dismiss first notification
      noti.at("6s").dismiss("notif_1");

      // Tap second notification (opens WhatsApp)
      noti.at("8s").tap("notif_2");

      // Show Dynamic Island after app opens
      noti.at("10s").dynamicIsland({
        mode: "compact",
        appId: "app_music",
      });

      // Third notification while in app
      noti.at("12s").show({
        id: "notif_3",
        appId: "app_whatsapp",
        title: "Alice 👋",
        body: "Hello? 👀",
        mode: "headsup",
      });

      // Swipe to dismiss
      noti.at("15s").swipe("notif_3", "left", "dismiss");

      // Clear all at end
      noti.at("18s").clearAll();
    },
  )

  // WhatsApp messages (match notifications)
  .track(
    "app_whatsapp",
    () => {
      return new WhatsAppTrackBuilder(30, "phone", "dm_alice", getOrder);
    },
    (wa) => {
      wa.at("1s").receive("Alice", "Hey! Are you free tonight?");
      wa.at("4s").receive("Alice", "Check out this photo! 📸");
      wa.at("12s").receive("Alice", "Hello? 👀");
    },
  )

  // Camera movements
  .camera((cam) => {
    cam.at("1s").focus("notification"); // Focus on banner
    cam.at("3s").animate({ scale: 1, duration: "0.5s" });
    cam.at("8s").animate({ scale: 1.1, duration: "0.5s" }); // Zoom on tap
  })

  .build();
```

## Advanced Features

### Notification Grouping

Stack related notifications using `groupKey`:

```typescript
noti.at("1s").show({
  id: "notif_1",
  appId: "app_whatsapp",
  title: "Alice",
  body: "Message 1",
  groupKey: "dm_alice", // Group notifications
});

noti.at("2s").show({
  id: "notif_2",
  appId: "app_whatsapp",
  title: "Alice",
  body: "Message 2",
  groupKey: "dm_alice", // Same group
});

// Shows as stacked notification: "Alice (2 messages)"
```

### Thread-Based Notifications

Use `threadId` for conversation threading:

```typescript
noti.at("1s").show({
  id: "notif_1",
  appId: "app_whatsapp",
  title: "Group Chat",
  body: "Alice: Hey everyone!",
  threadId: "conv_group_123",
});

noti.at("3s").show({
  id: "notif_2",
  appId: "app_whatsapp",
  title: "Group Chat",
  body: "Bob: Hi Alice!",
  threadId: "conv_group_123", // Same thread
});
```

### Notification Sounds

Notifications automatically trigger sounds via the audio system:

```typescript
noti.at("2s").show({
  appId: "app_whatsapp",
  title: "Alice",
  body: "Hey!",
  priority: "high", // Plays default notification sound
});
```

Sound is determined by:

- **Priority level** (urgent = longer sound)
- **App ID** (custom app sounds if registered)
- **Device profile** (iOS vs Android tones)

## Platform Differences

### iOS Behavior

- **Heads-up**: Banner slides down from top
- **Lockscreen**: Shows on lock screen with blur
- **Dynamic Island**: Supported on iPhone 14 Pro+
- **Notification Center**: Swipe down from top
- **Sounds**: iOS notification tones

### Android Behavior

- **Heads-up**: Card drops from status bar
- **Lockscreen**: Shows on ambient display
- **Dynamic Island**: Not supported
- **Notification Shade**: Swipe down from top
- **Sounds**: Android notification sounds

The DSL is platform-agnostic; rendering adapts to device profile.

## Best Practices

### Timing

- **Stagger notifications**: Wait 2-3s between notifications
- **Auto-dismiss**: Heads-up banners dismiss after 3-5s naturally
- **Tap timing**: Allow 1-2s after banner appears before tap
- **Sound overlap**: Avoid multiple notifications at exact same frame

### UX

- **Match app events**: Sync notification content with app messages
- **Use previews**: Rich previews make notifications more engaging
- **Group smartly**: Group related notifications, don't spam
- **Respect priority**: Use `urgent` sparingly for critical alerts

### Performance

- **Limit active notifications**: 10-15 max in notification center
- **Clear old notifications**: Use `clearAll()` to reset state
- **Optimize images**: Compress preview images for performance

## Troubleshooting

### Notification not appearing

- **Check mode**: `mode: "silent"` won't show banner
- **Check timing**: Ensure frame timing is correct
- **Check device state**: Device must be in correct state (locked/unlocked)
- **Check plugin registration**: Ensure `registerNotificationPlugin()` was called

### Notification stuck on screen

- **Auto-dismiss**: Heads-up notifications auto-dismiss after duration
- **Manual dismiss**: Call `.dismiss(id)` to remove
- **Clear all**: Use `.clearAll()` to reset

### Sound not playing

- **Check priority**: `priority: "low"` may not trigger sound
- **Check audio registry**: Ensure notification sounds are registered
- **Check device profile**: Some devices may have custom sound rules
