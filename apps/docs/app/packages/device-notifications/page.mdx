# @tokovo/device-notifications

> Enterprise-grade notification system for Tokovo phone simulations.

---

## Overview

`@tokovo/device-notifications` is a **P3 priority plugin** that brings OS-level notification functionality to Tokovo episodes. Display push notifications, banner alerts, notification centers, badges, and Dynamic Island interactions with a fluent DSL.

**Priority**: P3 (optional device feature)  
**Category**: Device Plugin  
**Platforms**: iOS, Android

---

## Installation

```bash
pnpm add @tokovo/device-notifications
```

---

## Quick Start

```typescript
import { NotificationTrackBuilder } from "@tokovo/device-notifications";
import { episode } from "@tokovo/dsl";

episode("story", { fps: 30, duration: "10s" })
  .device("phone", "iphone16", {
    app: "app_whatsapp",
  })
  .track(
    "notifications",
    (getOrder) => new NotificationTrackBuilder(30, "phone", getOrder),
    (noti) => {
      noti.at("2s").show({
        id: "notif_1",
        appId: "app_whatsapp",
        title: "Alice",
        body: "Hey! Are you free tonight?",
        mode: "headsup",
        priority: "high",
      });

      noti.at("5s").tap("notif_1");
      noti.at("8s").dismiss("notif_1");
    },
  )
  .build();
```

## Features

### Core Notification Types

| Type           | Description              | Behavior                                   |
| -------------- | ------------------------ | ------------------------------------------ |
| **Heads-Up**   | Banner at top of screen  | Auto-dismisses after 3-5s, can tap to open |
| **Lockscreen** | Shows on locked device   | Persists until dismissed or unlocked       |
| **Silent**     | No banner, goes to shade | Only visible in notification center        |
| **Badge**      | App icon badge only      | Number indicator, no banner                |

### Notification Behaviors

- **Push notifications** - Incoming message/alert display
- **Banner notifications** - Temporary top-of-screen alerts
- **Stacking** - Newest notification on top, older ones push down
- **Notification center** - Persistent notification tray
- **Notification badges** - App icon number indicators
- **Notification sounds** - Audio alerts (iOS/Android sounds)
- **Notification interactions** - Tap, swipe, dismiss actions
- **Dynamic Island** - iOS 16+ live activity integration

## Token System

The notification system uses platform-specific design tokens for consistent, native-feeling UI.

### Getting Tokens

```typescript
import {
  getNotificationTokens,
  createDeviceAwareTokens,
} from "@tokovo/device-notifications";

// Basic usage (static margins)
const tokens = getNotificationTokens("ios", "light");

// Device-aware (dynamic safe area)
const tokens = createDeviceAwareTokens({
  platform: "ios",
  theme: "light",
  safeArea: deviceProfile.safeArea, // { top: 110, bottom: 102, ... }
});
```

### Token Structure

```typescript
interface NotificationTokens {
  platform: "ios" | "android";
  theme: "light" | "dark";

  banner: {
    background: string;
    blur: number;
    borderRadius: number;
    padding: { top: number; bottom: number; horizontal: number };
    margin: { top: number; horizontal: number };
    gap: number;
    maxWidth: number;
  };

  text: {
    title: { color: string; fontSize: number; fontWeight: number };
    body: { color: string; fontSize: number };
    timestamp: { color: string; fontSize: number };
  };

  animation: {
    enterDuration: number;
    exitDuration: number;
    curve: string;
    enterTranslateY: number;
    exitTranslateY: number;
    enterScale: number;
    stackScaleDecay: number;
    stackSpacing: number;
  };
}
```

### Platform Differences

| Property                    | iOS | Android   |
| --------------------------- | --- | --------- |
| `banner.borderRadius`       | 38  | 24        |
| `banner.blur`               | 25  | 0 (solid) |
| `animation.enterTranslateY` | -50 | -40       |
| `animation.stackSpacing`    | 8   | 6         |
| `text.title.fontWeight`     | 600 | 500       |

## Stacking Behavior

When multiple notifications are active, they stack with **newest on top**:

```
┌─────────────────────────────────┐  ← Newest (stackIndex: 0)
│  Mom ❤️                         │
│  Call me when you get a chance  │
└─────────────────────────────────┘
   ┌─────────────────────────────┐  ← Older (stackIndex: 1)
   │  Sarah                      │     scale: 0.98, offset: 8px
   │  Did you see the news?      │
   └─────────────────────────────┘
      ┌──────────────────────────┐  ← Oldest (stackIndex: 2)
      │  Work Group              │     scale: 0.96, offset: 16px
      │  The deadline is tomorrow│
      └──────────────────────────┘
```

### Stack Calculation

```typescript
// For each notification in the stack:
const stackOffset = stackIndex * tokens.animation.stackSpacing;
const scale = 1 - stackIndex * tokens.animation.stackScaleDecay;
const zIndex = 9999 - stackIndex;
```

### Push-Down Animation

When a new notification arrives:

1. New notification enters at `stackIndex: 0`
2. Existing notifications increment their `stackIndex`
3. CSS transitions animate the position change smoothly

## Notification DSL

### NotificationTrackBuilder

The main entry point for authoring notification events.

```typescript
const noti = new NotificationTrackBuilder(fps, deviceId, getOrder);

noti.at("2s").show({ ... });
noti.at("5s").dismiss("notif_1");
noti.at("7s").tap("notif_2");
```

### Time Positioning

```typescript
noti.at("2s").show({ ... });      // 2 seconds
noti.at("500ms").show({ ... });   // 500 milliseconds
noti.at("30f").show({ ... });     // 30 frames
noti.at(60).show({ ... });        // Frame 60 (number)
```

## Notification Methods

### .show(opts)

Display a notification.

```typescript
noti.at("2s").show({
  id: "notif_1",
  appId: "app_whatsapp",
  title: "Alice",
  body: "Hey there!",
  mode: "headsup",
  priority: "high",
  icon: "/avatars/alice.jpg",
  duration: 5000,
  groupKey: "dm_alice",
  threadKey: "conv_123",
  replyable: true,
  actions: [
    { id: "reply", label: "Reply" },
    { id: "archive", label: "Archive", destructive: false },
  ],
  metadata: { conversationId: "123" },
});
```

### .dismiss(id)

Dismiss a notification.

```typescript
noti.at("5s").dismiss("notif_1");
```

### .tap(id, actionId?)

Tap a notification to open the app.

```typescript
noti.at("3s").tap("notif_1");
noti.at("3s").tap("notif_1", "reply"); // Tap action button
```

### .swipe(id, direction)

Swipe notification to dismiss.

```typescript
noti.at("4s").swipe("notif_1", "left");
noti.at("4s").swipe("notif_1", "right");
```

### .dynamicIsland(opts)

Update Dynamic Island state (iOS 16+).

```typescript
noti.at("2s").dynamicIsland({
  mode: "compact", // "idle" | "minimal" | "compact" | "expanded"
  appId: "app_music",
});
```

### .openPanel() / .closePanel()

Open/close notification shade.

```typescript
noti.at("5s").openPanel();
noti.at("8s").closePanel();
```

### .clearAll()

Clear all notifications.

```typescript
noti.at("10s").clearAll();
```

### .reply(id, text)

Send inline reply to notification.

```typescript
noti.at("6s").reply("notif_1", "Thanks!");
```

## Notification Properties

### Required Properties

| Property | Type     | Description                          |
| -------- | -------- | ------------------------------------ |
| `id`     | `string` | Unique notification identifier       |
| `appId`  | `string` | App that sent the notification       |
| `title`  | `string` | Notification title (sender/app name) |
| `body`   | `string` | Notification message text            |

### Optional Properties

| Property    | Type                   | Default     | Description                                      |
| ----------- | ---------------------- | ----------- | ------------------------------------------------ |
| `mode`      | `NotificationMode`     | `"headsup"` | Display mode (headsup, lockscreen, both, silent) |
| `priority`  | `NotificationPriority` | `"default"` | Priority level (low, default, high, critical)    |
| `icon`      | `string`               | App default | App icon URL or emoji                            |
| `duration`  | `number`               | 5000        | Auto-dismiss duration in ms                      |
| `actions`   | `Action[]`             | -           | Quick action buttons                             |
| `groupKey`  | `string`               | -           | Group notifications together                     |
| `threadKey` | `string`               | -           | Conversation thread ID                           |
| `replyable` | `boolean`              | `false`     | Allow inline reply                               |
| `metadata`  | `Record<string, any>`  | -           | App-specific data                                |

### NotificationMode

```typescript
type NotificationMode =
  | "headsup" // Banner + shade
  | "lockscreen" // Lockscreen only
  | "both" // Banner + lockscreen + shade
  | "silent"; // Shade only (no banner)
```

### NotificationPriority

```typescript
type NotificationPriority =
  | "low" // Minimal, no sound
  | "default" // Standard notification
  | "high" // Heads-up, sound
  | "critical"; // Full-screen alert, persistent
```

### Quick Actions


## Enterprise Integration

### NotificationAdapter Pattern

Apps customize their notification appearance and behavior by implementing `NotificationAdapter`:

```typescript
import type { NotificationAdapter } from "@tokovo/core";

export const whatsappNotificationAdapter: NotificationAdapter = {
  appId: "app_whatsapp",

  format(notification) {
    return {
      ...notification,
      icon: notification.senderAvatar ?? WHATSAPP_ICON,
      accentColor: "#25D366",
      actionButtons: [
        { id: "reply", label: "Reply", style: "primary" },
        { id: "mark_read", label: "Mark as Read" },
      ],
    };
  },

  handleAction(actionId, notification) {
    switch (actionId) {
      case "reply":
        return [
          { kind: "DEVICE", type: "OPEN_APP", appId: "app_whatsapp" },
          {
            kind: "APP",
            type: "NAVIGATE",
            route: `/chat/${notification.threadKey}`,
          },
        ];
      case "mark_read":
        return [
          {
            kind: "APP",
            type: "MARK_READ",
            conversationId: notification.threadKey,
          },
        ];
      default:
        return [];
    }
  },
};
```

### Plugin Contract Integration

Register the adapter in your plugin contract:

```typescript
import type { TokovoPluginContract } from "@tokovo/core";

export const WhatsAppPlugin: TokovoPluginContract<"app_whatsapp"> = {
  id: "app_whatsapp",
  version: "2.0.0",
  displayName: "WhatsApp",

  reducer: whatsappReducer,
  views: { AppRoot: WhatsAppApp },

  notificationAdapter: whatsappNotificationAdapter,

  dsl: WhatsAppDSL,
};
```

### Reducer-Based Notifications

Apps can emit notifications from their reducer:

```typescript
case "MESSAGE_RECEIVED": {
  const { conversationId, from, text } = action.payload;

  draft.conversations[conversationId].messages.push({
    from,
    text,
    at: action.at,
  });

  if (!draft.isInForeground) {
    draft._pendingEvents.push({
      kind: "DEVICE",
      type: "SHOW_NOTIFICATION",
      payload: {
        kind: "show",
        id: `notif_${Date.now()}`,
        appId: "app_whatsapp",
        title: from,
        body: text,
        priority: "high",
        threadKey: conversationId,
      },
    });
  }
  break;
}
```

### Integration Flow

```
Episode Event (MESSAGE_RECEIVED)
        │
        ▼
  App Reducer (WhatsApp)
        │
        ├── Updates conversation state
        │
        └── Emits SHOW_NOTIFICATION (if app not foreground)
                │
                ▼
        Device Notification Reducer
                │
                ├── Adds to notifications[]
                │
                └── Triggers banner display
                        │
                        ▼
                NotificationAdapter.format()
                        │
                        └── Returns app-branded appearance
                                │
                                ▼
                        NotificationBanner renders
                                │
                                ▼
                        User taps → handleAction()
                                │
                                └── Returns OPEN_APP + NAVIGATE events
```

## Platform Differences

### iOS Behavior

- **Heads-up**: Banner slides down from top with blur
- **Lockscreen**: Shows on lock screen with depth blur
- **Dynamic Island**: Supported on iPhone 14 Pro+
- **Notification Center**: Swipe down from top
- **Sounds**: iOS notification tones

### Android Behavior

- **Heads-up**: Card drops from status bar (Material Design)
- **Lockscreen**: Shows on ambient display
- **Dynamic Island**: Not supported (ignored)
- **Notification Shade**: Swipe down from top
- **Sounds**: Android notification sounds

The DSL is platform-agnostic; rendering adapts via the token system.

## Troubleshooting

### Notification not appearing

- **Check mode**: `mode: "silent"` won't show banner
- **Check timing**: Ensure frame timing is correct
- **Check priority**: Lowercase values (`"high"` not `"HIGH"`)

### Notification positioned incorrectly

- **Use device-aware tokens**: `createDeviceAwareTokens({ safeArea: ... })`
- **Check device profile**: Ensure correct `safeArea.top` value

### Stacking not working

- **Check sort direction**: Banner uses `sortDirection: "desc"` (newest first)
- **Verify IDs are unique**: Each notification needs a unique `id`

### Notification stuck on screen

- **Auto-dismiss**: Set `duration` for auto-dismissal
- **Manual dismiss**: Call `.dismiss(id)` to remove
- **Clear all**: Use `.clearAll()` to reset
