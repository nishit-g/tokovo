# @tokovo/dsl

> **Priority: P1** — The primary TypeScript fluent API for authoring episodes in a developer-friendly format.

`@tokovo/dsl` is the **Domain-Specific Language** that makes episode authoring natural, type-safe, and enjoyable. Instead of writing raw IR JSON, you use a fluent TypeScript API that feels like storytelling.

## Installation

```bash
npm install @tokovo/dsl
# or
pnpm add @tokovo/dsl
```

## What is @tokovo/dsl?

DSL is the **developer-facing authoring layer** for Tokovo episodes. It provides:

- ✅ **Type safety** — TypeScript catches errors before runtime
- ✅ **Autocomplete** — IDE suggestions guide you through the API
- ✅ **Readability** — Code reads like a narrative screenplay
- ✅ **Composability** — Reusable functions and patterns
- ✅ **Validation** — Compile-time checks prevent invalid episodes
- ✅ **Fluent API** — Method chaining for expressive authoring

**DSL is how developers write episodes.** It compiles to IR JSON for execution.

## Why DSL Instead of Raw IR?

Compare these two approaches to creating the same episode:

### DSL (TypeScript)

```typescript
import { episode } from "@tokovo/dsl";
import { WhatsAppTrackBuilder } from "@tokovo/apps-whatsapp";

let order = 0;
const getOrder = () => order++;

episode("story", { fps: 30, duration: "10s" })
  .device("phone", "iphone16", {
    app: "app_whatsapp",
    conversations: [{ id: "dm_alice", name: "Alice" }],
  })
  .track(
    "app_whatsapp",
    () => new WhatsAppTrackBuilder(30, "phone", "dm_alice", getOrder),
    (wa) => {
      wa.at("1s").receive("Alice", "Hey!");
      wa.at("3s").send("Hi there!");
    },
  )
  .camera((cam) => {
    cam.at("1s").animate({ scale: 1.2, duration: "0.5s" });
  })
  .build();
```

### Raw IR (JSON)

```json
{
  "id": "story",
  "fps": 30,
  "durationInFrames": 300,
  "devices": [
    {
      "id": "phone",
      "profile": "iphone16",
      "app": "app_whatsapp",
      "conversations": [{ "id": "dm_alice", "name": "Alice" }]
    }
  ],
  "events": [
    {
      "at": 30,
      "kind": "APP",
      "deviceId": "phone",
      "appId": "app_whatsapp",
      "type": "MESSAGE_RECEIVED",
      "payload": {
        "conversationId": "dm_alice",
        "from": "Alice",
        "content": "Hey!"
      }
    },
    {
      "at": 90,
      "kind": "APP",
      "deviceId": "phone",
      "appId": "app_whatsapp",
      "type": "MESSAGE_SENT",
      "payload": {
        "conversationId": "dm_alice",
        "content": "Hi there!"
      }
    },
    {
      "at": 30,
      "kind": "CAMERA",
      "type": "ANIMATE_START",
      "duration": 15,
      "payload": { "scale": 1.2, "easing": "linear" }
    },
    {
      "at": 45,
      "kind": "CAMERA",
      "type": "ANIMATE_END",
      "payload": {}
    }
  ],
  "markers": [],
  "sections": []
}
```

### Benefits of DSL

1. **Type Safety** — TypeScript catches typos like `wa.at('1s').recieve()` (invalid method)
2. **Autocomplete** — IDE shows available methods: `.receive()`, `.send()`, `.typing()`
3. **Readability** — `.at('1s').receive('Alice', 'Hey!')` is self-documenting
4. **Time Parsing** — Write `'1s'` instead of calculating `30` frames
5. **Event Expansion** — `.animate()` creates both START and END events automatically
6. **Validation** — Compile-time errors prevent invalid episodes

## Core Concepts

### 1. Episode Builder

The `episode()` function creates an episode builder:

```typescript
import { episode } from "@tokovo/dsl";

const ir = episode("my-story", {
  fps: 30, // Frames per second
  duration: "30s", // Episode length
  title: "My Story", // Display title
  description: "A tale of...",
})
  // ... chain methods here
  .build(); // Compile to IR
```

**Required parameters:**

- `id` — Unique episode identifier (kebab-case recommended)
- `fps` — Frame rate (typically 30 or 60)
- `duration` — Total length (e.g., `'30s'`, `'1.5min'`, or frame count)

**Optional parameters:**

- `title` — Human-readable title
- `description` — What the episode shows

### 2. Fluent API Pattern

The DSL uses **method chaining** for expressive authoring:

```typescript
episode("story", { fps: 30, duration: "10s" })
  .device("phone", "iphone16", { app: "app_whatsapp" })
  .camera((cam) => cam.at("0s").set({ scale: 1 }))
  .audio((audio) => audio.span("0s", "10s").bgm("track"))
  .build();
```

Each method returns `this`, allowing continuous chaining.

### 3. Track-Based Architecture

Episodes are composed of **tracks** (like video editing software):

- **Camera Track** — Camera movements and effects
- **Audio Track** — Background music and sound effects
- **OS Track** — Device state (battery, time, notifications)
- **App Tracks** — App-specific events (WhatsApp messages, etc.)

Each track has its own builder with scoped methods.

### 4. Time Parsing

Write times naturally instead of calculating frames:

```typescript
cam.at("1s"); // 30 frames at 30fps
cam.at("1.5s"); // 45 frames
cam.at("0.5min"); // 900 frames
cam.at(90); // Explicit frame number
```

Durations work the same way:

```typescript
cam.animate({ scale: 1.2, duration: "0.5s" });
audio.span("0s", "30s").bgm("track");
```

## Episode API

### `episode(id, config)`

Creates a new episode builder.

```typescript
episode(
  id: string,
  config: {
    fps: number,
    duration: string | number,
    title?: string,
    description?: string
  }
): EpisodeBuilder
```

**Example:**

```typescript
episode("demo", {
  fps: 30,
  duration: "30s",
  title: "Demo Episode",
  description: "Shows basic features",
});
```

### `.device(id, profile, options)`

Add a device to the episode.

```typescript
.device(
  id: string,
  profile: string,
  options: {
    app: string,
    conversations?: ConversationConfig[],
    os?: OSConfig
  }
): this
```

**Example:**

```typescript
.device('phone', 'iphone16', {
  app: 'app_whatsapp',
  conversations: [
    { id: 'dm_alice', name: 'Alice', avatar: '/avatars/alice.jpg' }
  ],
  os: {
    time: new Date('2024-12-17T19:30:00'),
    battery: 85,
    network: '5G'
  }
})
```

**Available profiles:**

- `'iphone16'` — iPhone 16 Pro
- `'iphone15'` — iPhone 15
- `'pixel8'` — Google Pixel 8
- More coming soon

### `.camera(fn)`

Add camera movements and effects.

```typescript
.camera(fn: (cam: CameraTrackBuilder) => void): this
```

**Example:**

```typescript
.camera(cam => {
  cam.at('0s').set({ scale: 1 });
  cam.at('1s').animate({ scale: 1.2, duration: '0.5s' });
  cam.at('3s').focus('lastMessage', { scale: 1.1 });
  cam.at('5s').shake({ intensityX: 5, intensityY: 5, duration: '0.3s' });
})
```

### `.audio(fn)`

Add background music and sound effects.

```typescript
.audio(fn: (audio: AudioTrackBuilder) => void): this
```

**Example:**

```typescript
.audio(audio => {
  audio.span('0s', '30s').bgm('romantic_theme', { volume: 0.3 });
  audio.at('10s').play('notification_sound');
  audio.at('20s').crossfade('tension_theme', { duration: '2s' });
})
```

### `.os(fn)`

Control device operating system state.

```typescript
.os(fn: (os: OSTrackBuilder) => void): this
```

**Example:**

```typescript
.os(os => {
  os.at('0s').set({ battery: 85, time: new Date(), network: '5G' });
  os.at('5s').battery(83);
  os.at('10s').notification({
    appId: 'app_whatsapp',
    title: 'Alice',
    body: 'Hey!'
  });
})
```

### `.track(appId, factory, fn)`

Add an app-specific track.

```typescript
.track<T>(
  appId: string,
  factory: () => T,
  fn: (track: T) => void
): this
```

**Example:**

```typescript
import { WhatsAppTrackBuilder } from '@tokovo/apps-whatsapp';

let order = 0;
const getOrder = () => order++;

.track('app_whatsapp',
  () => new WhatsAppTrackBuilder(30, 'phone', 'dm_alice', getOrder),
  wa => {
    wa.at('1s').receive('Alice', 'Hey!');
    wa.at('3s').send('Hi there!');
  }
)
```

### `.mark(id, time)` and `.section(id, start, end)`

Add markers and sections for navigation.

```typescript
.mark(id: string, time: string | number): this
.section(id: string, start: string | number, end: string | number): this
```

**Example:**

```typescript
.mark('intro', '0s')
.mark('climax', '15s')
.section('setup', '0s', '10s')
.section('conflict', '10s', '20s')
```

### `.build()`

Compile the episode to IR.

```typescript
.build(): TrackEpisodeIR
```

**Example:**

```typescript
const ir = episode("story", { fps: 30, duration: "10s" })
  .device("phone", "iphone16", { app: "app_whatsapp" })
  .build();

// ir is now a TrackEpisodeIR object (JSON-serializable)
```

## Camera Track API

### Point Operations (`.at(time)`)

Operations that happen instantly at a specific time.

#### `.set(options)`

Set camera state instantly.

```typescript
cam.at('0s').set({
  x?: number,
  y?: number,
  scale?: number,
  rotation?: number,
  originX?: number,
  originY?: number
})
```

**Example:**

```typescript
cam.at("0s").set({ scale: 1, x: 0, y: 0 });
```

#### `.animate(options)`

Animate camera to new state.

```typescript
cam.at('1s').animate({
  x?: number,
  y?: number,
  scale?: number,
  rotation?: number,
  duration: string | number,
  easing?: 'linear' | 'easeIn' | 'easeOut' | 'easeInOut' | 'cinematic'
})
```

**Example:**

```typescript
cam.at("1s").animate({
  scale: 1.2,
  duration: "0.5s",
  easing: "cinematic",
});
```

#### `.focus(anchorId, options)`

Focus camera on an element.

```typescript
cam.at('2s').focus('lastMessage', {
  scale?: number,
  padding?: number,
  duration?: string | number,
  easing?: EasingType
})
```

**Example:**

```typescript
cam.at("3s").focus("lastMessage", { scale: 1.15, duration: "0.4s" });
```

#### `.shake(options)`

Apply screen shake effect.

```typescript
cam.at('5s').shake({
  intensityX: number,
  intensityY: number,
  frequency?: number,
  decay?: number,
  duration: string | number
})
```

**Example:**

```typescript
cam.at("5s").shake({
  intensityX: 5,
  intensityY: 4,
  duration: "0.3s",
});
```

#### `.reset(options)`

Reset camera to default state.

```typescript
cam.at('10s').reset({
  duration?: string | number,
  easing?: EasingType
})
```

**Example:**

```typescript
cam.at("10s").reset({ duration: "1s", easing: "easeOut" });
```

### Span Operations (`.span(start, end)`)

Operations that last for a duration.

#### `.track(anchorId, options)`

Continuously track an element.

```typescript
cam.span('5s', '10s').track('inputArea', {
  scale?: number,
  lag?: number
})
```

**Example:**

```typescript
cam.span("5s", "10s").track("typingIndicator", { scale: 1.1 });
```

## Audio Track API

### Point Operations (`.at(time)`)

#### `.play(soundId, options)`

Play a one-shot sound effect.

```typescript
audio.at('2s').play('notification', {
  volume?: number,
  loop?: boolean
})
```

#### `.stop(soundId)` / `.stopAll()`

Stop specific sound or all sounds.

```typescript
audio.at("5s").stop("notification");
audio.at("10s").stopAll();
```

#### `.crossfade(soundId, options)`

Crossfade to a new track.

```typescript
audio.at('15s').crossfade('tension_theme', {
  duration: string | number,
  volume?: number
})
```

#### `.fadeOut(options)`

Fade out all audio.

```typescript
audio.at("28s").fadeOut({ duration: "2s" });
```

### Span Operations (`.span(start, end)`)

#### `.bgm(soundId, options)`

Play background music for duration.

```typescript
audio.span('0s', '30s').bgm('romantic_theme', {
  volume?: number,
  fadeIn?: string | number,
  fadeOut?: string | number
})
```

**Example:**

```typescript
audio.span("0s", "30s").bgm("theme", {
  volume: 0.3,
  fadeIn: "3s",
  fadeOut: "4s",
});
```

## OS Track API

### Point Operations (`.at(time)`)

#### `.set(options)`

Set full OS state.

```typescript
os.at('0s').set({
  time?: Date | number,
  battery?: number,
  charging?: boolean,
  network?: 'wifi' | '5G' | '4G' | '3G' | 'none',
  strength?: number,
  dnd?: boolean,
  lowPowerMode?: boolean
})
```

#### `.time(date)`

Set device time.

```typescript
os.at("0s").time(new Date("2024-12-17T19:30:00"));
```

#### `.battery(level, options)`

Set battery level.

```typescript
os.at("5s").battery(83, { charging: false });
```

#### `.network(type, options)`

Set network status.

```typescript
os.at("10s").network("5G", { strength: 4 });
```

#### `.notification(options)`

Show a notification.

```typescript
os.at('2s').notification({
  appId: string,
  title: string,
  body: string,
  icon?: string,
  mode?: 'headsup' | 'lockscreen' | 'both'
})
```

#### `.dismissNotification(id)` / `.dismissAllNotifications()`

Dismiss notifications.

```typescript
os.at("5s").dismissNotification("notif_1");
os.at("10s").dismissAllNotifications();
```

## Compilation

The DSL compiles to IR (Intermediate Representation) JSON:

```typescript
import { episode } from "@tokovo/dsl";
import { prepareTrackEpisode } from "@tokovo/compiler";
import { whatsappPlugin } from "@tokovo/apps-whatsapp";

// 1. Author episode with DSL
const ir = episode("story", { fps: 30, duration: "10s" })
  .device("phone", "iphone16", { app: "app_whatsapp" })
  .build();

// 2. Compile to executable format
const prepared = prepareTrackEpisode(ir, [whatsappPlugin]);

// 3. Use in renderer
export const MyComposition = () => runEpisode(prepared);
```

**Compilation process:**

1. **DSL → IR** — `.build()` produces `TrackEpisodeIR` JSON
2. **IR → RuntimeEvents** — `@tokovo/compiler` lowers track events
3. **Execution** — `@tokovo/core` replays events frame-by-frame

## Complete Example

```typescript
import { episode } from "@tokovo/dsl";
import { WhatsAppTrackBuilder } from "@tokovo/apps-whatsapp";

let order = 0;
const getOrder = () => order++;

const ir = episode("breakup-scene", {
  fps: 30,
  duration: "45s",
  title: "Breakup Scene",
  description: "A tense WhatsApp conversation",
})
  // Configure device
  .device("phone", "iphone16", {
    app: "app_whatsapp",
    conversations: [
      { id: "dm_alex", name: "Alex", avatar: "/avatars/alex.jpg" },
    ],
    os: {
      time: new Date("2024-12-17T23:30:00"),
      battery: 15,
      network: "wifi",
    },
  })

  // WhatsApp conversation
  .track(
    "app_whatsapp",
    () => new WhatsAppTrackBuilder(30, "phone", "dm_alex", getOrder),
    (wa) => {
      wa.at("1s").receive("Alex", "We need to talk.");
      wa.span("3s", "6s").typing("me");
      wa.at("6s").send("About what?");
      wa.span("8s", "12s").typing("them");
      wa.at("12s").receive("Alex", "I don't think this is working...");
      wa.span("15s", "18s").typing("me");
      wa.at("18s").send("Are you serious?");
      wa.span("20s", "25s").typing("them");
      wa.at("25s").receive("Alex", "I'm sorry. I need space.");
      wa.at("30s").read();
    },
  )

  // Camera movements
  .camera((cam) => {
    cam.at("0s").set({ scale: 1 });
    cam.at("1s").animate({ scale: 1.1, duration: "0.5s", easing: "easeOut" });
    cam.at("12s").focus("lastMessage", { scale: 1.2, duration: "0.8s" });
    cam.at("12.5s").shake({ intensityX: 3, intensityY: 3, duration: "0.4s" });
    cam
      .at("25s")
      .animate({ scale: 1.3, y: -30, duration: "1s", easing: "cinematic" });
    cam.at("40s").reset({ duration: "2s" });
  })

  // Audio
  .audio((audio) => {
    audio.span("0s", "45s").bgm("tension", { volume: 0.15, fadeOut: "5s" });
    audio.at("12s").play("heartbeat", { volume: 0.3 });
    audio.at("25s").crossfade("somber", { duration: "3s", volume: 0.2 });
  })

  // OS state changes
  .os((os) => {
    os.at("15s").battery(14);
    os.at("30s").battery(13);
  })

  // Navigation markers
  .mark("start", "0s")
  .mark("tension", "12s")
  .mark("climax", "25s")
  .section("setup", "0s", "12s")
  .section("conflict", "12s", "30s")
  .section("aftermath", "30s", "45s")

  .build();

// ir is now ready for compilation and execution
```

## Best Practices

### 1. Use Declaration Order Counters

Always use order counters for app tracks:

```typescript
let order = 0;
const getOrder = () => order++;

.track('app_whatsapp',
  () => new WhatsAppTrackBuilder(30, 'phone', 'dm_alice', getOrder),
  wa => { ... }
)
```

This ensures events at the same frame execute in declaration order.

### 2. Write Times Naturally

Prefer string times over frame numbers:

```typescript
// ✅ Good
cam.at("1.5s").animate({ scale: 1.2, duration: "0.5s" });

// ❌ Avoid (unless you need precise frame control)
cam.at(45).animate({ scale: 1.2, duration: 15 });
```

### 3. Extract Common Patterns

Reuse builders for consistency:

```typescript
function typingDelay(wa: WhatsAppTrackBuilder, start: string, end: string) {
  wa.span(start, end).typing("them");
}

typingDelay(wa, "3s", "5s");
typingDelay(wa, "8s", "11s");
```

### 4. Use Markers for Navigation

Add markers to help debugging:

```typescript
.mark('intro', '0s')
.mark('reveal', '15s')
.mark('finale', '28s')
```

### 5. Keep Episodes Under 60s

Shorter episodes are easier to author, debug, and render. Chain multiple episodes for longer content.
