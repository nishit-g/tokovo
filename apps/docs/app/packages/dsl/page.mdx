# @tokovo/dsl

The TypeScript authoring API for Tokovo episodes.

## Responsibilities

- episode builders
- device bootstrap authoring
- app snapshot and initial-view authoring
- camera and audio choreography entrypoints

Use `@tokovo/dsl` for scene timing and sequencing.

## Role In The Stack

`@tokovo/dsl` is the primary authoring surface before compilation. It should stay expressive and deterministic without turning into a CMS abstraction layer.

Bootstrap rule:

- `device()` is only for device/OS state
- `snapshot()` is only for plugin-owned previous app data
- `view()` is only for explicit frame-0 app routing
