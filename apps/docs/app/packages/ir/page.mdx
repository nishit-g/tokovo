# @tokovo/ir

> **Priority: P1** — The canonical JSON format for episodes. The single source of truth.

`@tokovo/ir` defines **Intermediate Representation (IR)** — the platform-independent, serializable JSON format that sits at the heart of Tokovo. IR is the bridge between authoring (DSL) and execution (engine).

## Installation

```bash
npm install @tokovo/ir
# or
pnpm add @tokovo/ir
```

## What is IR (Intermediate Representation)?

IR is the **canonical JSON format** for episodes. It's designed to be:

- ✅ **Platform-independent** — Not tied to TypeScript/JavaScript
- ✅ **Serializable** — Can be saved to `.json` files, transmitted over networks
- ✅ **Human-readable** — JSON structure that's easy to inspect
- ✅ **Machine-parseable** — Strict schemas with Zod validation
- ✅ **AI-friendly** — LLMs can generate valid IR directly
- ✅ **Git-friendly** — Version control episodes as pure data

**IR is the single source of truth** for what happens in an episode.

---

## Why IR Matters

IR sits at the center of Tokovo's architecture:

```
┌──────────────┐
│     DSL      │  TypeScript authoring (episode builder)
│ (authoring)  │
└──────┬───────┘
       │
       ▼ COMPILER
┌──────────────┐
│      IR      │  Platform-independent JSON
│ (canonical)  │  ← Single source of truth
└──────┬───────┘
       │
       ▼ ENGINE
┌──────────────┐
│ WorldState   │  Frame-by-frame rendering
│ (execution)  │
└──────────────┘
```

### Key Benefits:

1. **DSL compiles to IR** — TypeScript DSL → IR JSON via `@tokovo/compiler`
2. **Engine reads IR** — `@tokovo/core`'s `replay()` takes IR as input
3. **Portability** — Save episodes as `.json` files, load them anywhere
4. **Version control** — Git-friendly JSON you can diff and review
5. **AI generation** — LLMs can create valid IR without knowing TypeScript
6. **Tool interop** — Other tools (editors, validators) can generate episodes

## TrackEvent Format

Every event in IR follows the **TrackEvent** schema:

```typescript
{
  at: number,             // Start frame (required)
  duration?: number,      // Duration in frames (for spans)
  deviceId?: string,      // Device routing
  kind: string,           // Event category
  type: string,           // Event type
  payload: object         // Event-specific data
}
```

### Event Categories (kind)

| Kind     | Purpose                      | Examples                       |
| -------- | ---------------------------- | ------------------------------ |
| `APP`    | App-specific events          | MESSAGE_RECEIVED, TYPING_START |
| `DEVICE` | Device-level operations      | LOCK, UNLOCK, OPEN_APP         |
| `CAMERA` | Camera movements             | ZOOM, FOCUS, SHAKE             |
| `AUDIO`  | Sound effects                | BGM_START, PLAY, CROSSFADE     |
| `VOICE`  | Voice-over segments          | PLAY_SEGMENT, STOP_VOICE       |
| `OS`     | Operating system state       | SET_BATTERY, SET_TIME          |
| `CALL`   | Phone call events            | INCOMING, ANSWER, END          |
| `MARKER` | Debugging/navigation markers | MARK, SECTION_START            |

### Example Events

**App Event (WhatsApp message):**

```json
{
  "at": 30,
  "kind": "APP",
  "deviceId": "phone",
  "appId": "app_whatsapp",
  "type": "MESSAGE_RECEIVED",
  "payload": {
    "conversationId": "conv_alice",
    "from": "Alice",
    "content": "Hey! How are you?"
  }
}
```

**Camera Event (zoom in):**

```json

    "scale": 1.5,
    "duration": 30,

```

**Device Event (unlock phone):**

```json

```

**Voice Event (play segment):**


## DSL vs IR: The Relationship

The **DSL** (Domain-Specific Language) is how you _author_ episodes. The **IR** is what gets executed.

### DSL (Authoring - TypeScript)

```typescript
import { episode } from "@tokovo/dsl";

episode("breakup-01", { duration: "10s" })
  .device("phone", "iphone16", (d) => {
    d.app("app_whatsapp", (wa) => {
      wa.conversation("conv_alice", "Alice", (c) => {
        c.at("1s").receive("Alice", "We need to talk.");
        c.at("3s").receive("Alice", "This isn't working.");
      });
    });
  })
  .camera((cam) => {
    cam.at("3s").zoom(1.2, "1s", "cinematic");
  })
  .build();
```

### ↓ COMPILER ↓

The `@tokovo/compiler` transforms this into:

### IR (Execution - JSON)

```json
{
  "id": "breakup-01",
  "fps": 30,
  "durationInFrames": 300,
  "devices": [
    {
      "id": "phone",
      "profile": "iphone16",
      "app": "app_whatsapp",
      "conversations": [{ "id": "conv_alice", "name": "Alice" }]
    }
  ],
  "events": [
    {
      "at": 30,
      "kind": "APP",
      "appId": "app_whatsapp",
      "type": "MESSAGE_RECEIVED",
      "payload": {
        "conversationId": "conv_alice",
        "from": "Alice",
        "content": "We need to talk."
      }
    },
    {
      "at": 90,
      "kind": "APP",
      "appId": "app_whatsapp",
      "type": "MESSAGE_RECEIVED",
      "payload": {
        "conversationId": "conv_alice",
        "from": "Alice",
        "content": "This isn't working."
      }
    },
    {
      "at": 90,
      "kind": "CAMERA",
      "type": "ZOOM",
      "payload": {
        "scale": 1.2,
        "duration": 30,
        "easing": "cinematic"
      }
    }
  ],
  "markers": [],
  "sections": []
}
```

**Why this separation?**

- DSL = **Developer ergonomics** (chainable API, type safety)
- IR = **Execution contract** (stable, validated, portable)

## Working with IR

### Loading IR from File

```typescript
import type { TrackEpisodeIR } from "@tokovo/ir";
import { validateTrackEpisodeIR } from "@tokovo/ir";
import { prepareEpisode } from "@tokovo/core";
import fs from "fs";

// Load IR JSON
const episodeJSON = JSON.parse(fs.readFileSync("episode.json", "utf-8"));

// Validate IR
const episodeIR: TrackEpisodeIR = validateTrackEpisodeIR(episodeJSON);

// Prepare for execution
const prepared = prepareEpisode(episodeIR);
// Returns: { initial, events, metadata }
```

### Safe Validation

```typescript
import { safeValidateTrackEpisodeIR } from "@tokovo/ir";

const result = safeValidateTrackEpisodeIR(episodeJSON);

if (result.success) {
  console.log("Valid IR:", result.data);
} else {
  console.error("Validation errors:", result.error);
}
```

### Executing IR

```typescript
import { replay } from "@tokovo/core";
import type { TrackEpisodeIR } from "@tokovo/ir";

const episodeIR: TrackEpisodeIR = JSON.parse(fs.readFileSync("episode.json"));

// Get world state at frame 90
const worldState = replay(episodeIR.initial, episodeIR.events, 90);

console.log(worldState.devices.phone.foregroundAppId);
// Output: 'app_whatsapp'
```

## Device Configuration

The `devices` array defines initial device state:

```typescript
{
  id: string              // Unique device identifier
  profile: string         // Device profile ('iphone16', 'pixel8')
  app: string             // Initial foreground app
  conversations?: []      // Pre-configured conversations
  os?: {                  // OS state
    time?: number
    battery?: number
    network?: 'wifi' | '5G' | '4G'
  }
  theme?: string          // UI theme/strategy
}
```

**Example:**

## Validation with Zod

IR uses **Zod schemas** for runtime validation:

```typescript
import { TrackEpisodeIRSchema } from "@tokovo/ir";

const result = TrackEpisodeIRSchema.safeParse(episodeJSON);

if (result.success) {
  // Type-safe IR
  const ir = result.data;
} else {
  // Detailed validation errors
  console.error(result.error.format());
}
```

**What gets validated:**

- ✅ Episode structure (id, fps, durationInFrames)
- ✅ Device configuration (profile, app, conversations)
- ✅ Event structure (at, kind, type, payload)
- ✅ Marker/section integrity (frame numbers)
- ✅ Type constraints (fps > 0, frames ≥ 0)

## Module Augmentation (Advanced)

Apps can extend IR types using **module augmentation**:

```typescript
// In apps-whatsapp/src/ir.ts
declare module "@tokovo/ir" {
  interface AppTrackEventRegistry {
    app_whatsapp: WhatsAppTrackEvent;
  }

  interface AppPayloadRegistry {
    app_whatsapp: WhatsAppPayloads;
  }
}

// Now WhatsApp events are part of TrackEvent union
export type WhatsAppTrackEvent = TrackEventBase & {
  kind: "APP";
  appId: "app_whatsapp";
} & (
    | {
        type: "MESSAGE_RECEIVED";
        payload: { conversationId: string; from: string; content: string };
      }
    | {
        type: "TYPING_START";
        payload: { conversationId: string; participant: string };
      }
    | {
        type: "TYPING_END";
        payload: { conversationId: string; participant: string };
      }
  );
```
