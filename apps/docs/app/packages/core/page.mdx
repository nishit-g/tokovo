# @tokovo/core

The deterministic runtime layer.

## Responsibilities

- world-state computation
- replay and runtime state transitions
- engine registries
- shared runtime types
- audio primitives
- plugin contracts used by compiler, renderer, and app packages

## Boundary

`@tokovo/core` must stay headless and should not depend on React or renderer code.

## Key Exports

- replay and runtime state: `replayIncremental`, `createEventIndex`, `createKeyframedEventIndex`, `createStateCache`
- track preparation: `prepareTrackEpisode` from `@tokovo/compiler`
- engine helpers: `createEventIndex`, `createKeyframedEventIndex`, `createStateCache`
- registries: `createEngine`, `createEngineRegistries`, `createReducerRegistry`
- anchors: `AnchorRegistryClass`, `createAnchorRegistry`
- config: `TokovoConfig`, `EngineConfig`
- validation and logging helpers
- shared runtime and event types

## Design Rule

If a feature can be expressed as a pure runtime transition, it belongs here. If it needs JSX,
device chrome, or visual composition, it belongs in renderer or a plugin UI layer instead.
