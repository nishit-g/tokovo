# @tokovo/core

> **Priority: P0** — The deterministic phone simulation engine at the heart of Tokovo.

`@tokovo/core` is the foundational package that powers Tokovo's deterministic phone simulation. It provides the pure state engine, plugin system, and runtime infrastructure that transforms timeline events into immersive phone UI videos.

## Installation

```bash
npm install @tokovo/core
# or
pnpm add @tokovo/core
```

## What is @tokovo/core?

`@tokovo/core` is **pure logic** — no React, no Remotion, no rendering. It defines:

- **Event-driven state management** — Transform timeline events into world state
- **Plugin architecture** — Self-contained apps register reducers, views, and assets
- **Type system** — WorldState, TimelineEvent, and 250+ TypeScript exports
- **Registries** — App, Sound, Layout, Anchor, and Widget registration systems
- **Deterministic replay** — Given the same events and initial state, output is identical

The core engine is framework-agnostic and optimized for:

- **AI-generated content** — LLMs generate JSON events, Tokovo renders videos
- **Deterministic rendering** — No side effects, fully reproducible
- **Extensibility** — Add devices, apps, and behaviors without modifying core

## Major Subsystems

### 1. Engine — Deterministic Replay

The core engine is a **pure function** that computes `WorldState` at any point in time:

```typescript
function replay(
  initial: WorldState,
  events: TimelineEvent[],
  t: number,
): WorldState;
```

**How it works:**

1. Filter events where `event.at <= t`
2. Apply events sequentially using registered reducers
3. Return immutable state snapshot

**Key features:**

- ✅ **Deterministic** — Same input always produces same output
- ✅ **Pure** — No side effects, no hidden state
- ✅ **Immer-based** — Clean immutable updates with structural sharing
- ✅ **Performance-optimized** — Caching, event indexing, incremental replay

**Example:**

```typescript
import { replay, createInitialWorld, createTokovoRegistries } from "@tokovo/core";

const initialWorld = createInitialWorld({
  devices: {
    main_phone: {
      id: "main_phone",
      profileId: "iphone16",
      isLocked: true,
      notifications: [],
    },
  },
});

const registries = createTokovoRegistries();

const registries = createTokovoRegistries();

const events = [
  { at: 0, kind: "OS", type: "UNLOCK" },
  { at: 30, kind: "OS", type: "OPEN_APP", payload: { appId: "app_whatsapp" } },
];

// Get state at frame 45
const worldAt45 = replay(initialWorld, events, 45, {
  mode: "preview",
  fps: 30,
  registries: registries.engine,
});
// worldAt45.devices.main_phone.isLocked === false
// worldAt45.devices.main_phone.foregroundAppId === 'app_whatsapp'
```

### 2. Registries — Centralized Lookup Systems

`@tokovo/core` provides **five core registries** for managing plugins and resources:

#### **App Registry**

Maps `appId` → React view component

```typescript
import { createTokovoRegistries } from "@tokovo/core";

const registries = createTokovoRegistries();
registries.plugins.apps.register("app_whatsapp", WhatsAppView);
const view = registries.plugins.apps.get("app_whatsapp");
```

#### **Sound Registry**

Manages sound asset URLs with namespacing

```typescript
import { createTokovoRegistries } from "@tokovo/core";

const registries = createTokovoRegistries();
registries.plugins.sounds.registerNamespaced("app_whatsapp", {
  notification: "/sounds/whatsapp-notification.mp3",
  sent: "/sounds/whatsapp-sent.mp3",
});

const url = registries.plugins.sounds.get("app_whatsapp:notification");
```

#### **Anchor Registry**

Resolves semantic positions (e.g., "message bubble", "send button")

```typescript
import { createTokovoRegistries } from "@tokovo/core";

const registries = createTokovoRegistries();
const bounds = registries.plugins.anchors.resolveAnchor(
  "app_whatsapp:send_button",
  worldState,
  "main_phone",
);
// Returns { x, y, width, height } in screen coordinates
```

#### **Layout Registry**

Platform-specific layout computation strategies

```typescript
import { createTokovoRegistries } from "@tokovo/core";

const registries = createTokovoRegistries();
registries.plugins.layouts.register({
  appId: "app_whatsapp",
  viewKind: "CHAT",
  platforms: ["ios", "android"],
  computeLayout: (ctx) => {
    // Compute bubble positions, scroll offset, etc.
    return layoutState;
  },
});
```

#### **Widget Registry**

Dynamic Island, status bar, and lockscreen widgets

```typescript
import { createTokovoRegistries } from "@tokovo/core";

const registries = createTokovoRegistries();
registries.plugins.widgets.register({
  appId: "app_music",
  mode: "dynamicIsland",
  platforms: ["ios"],
  component: MusicNowPlayingWidget,
});
```

### 3. Audio System — Enterprise-Grade Sound

`@tokovo/core` includes a complete audio subsystem for deterministic sound playback, mixing, and automation.

#### Audio Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                       AUDIO SYSTEM                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐  ┌───────────┐ │
│  │   Buses    │  │  Policies  │  │   Mixer    │  │  Handlers │ │
│  │            │  │            │  │            │  │           │ │
│  │ music/ui/  │  │ spam gate  │  │ gain calc  │  │ PLAY/STOP │ │
│  │ sfx/voice  │  │ concurrency│  │ ducking    │  │ CROSSFADE │ │
│  └────────────┘  └────────────┘  └────────────┘  └───────────┘ │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │                        AudioState                          │ │
│  │  activeSounds: Record<string, SoundCue>                    │ │
│  │  musicBed?: MusicBed                                       │ │
│  │  outgoingMusicBed?: MusicBed  (for crossfade)             │ │
│  │  buses: { music, ui, sfx, voice }                         │ │
│  └────────────────────────────────────────────────────────────┘ │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### Audio Buses

Sounds are routed through specialized buses with independent gain and concurrency limits:

| Bus     | Purpose                           | Max Concurrent | Default Gain |
| ------- | --------------------------------- | -------------- | ------------ |
| `music` | Background music, BGM tracks      | 1              | 0.35         |
| `ui`    | UI feedback sounds (taps, swipes) | 3              | 0.9          |
| `sfx`   | Sound effects                     | 4              | 0.8          |
| `voice` | Voice-over narration              | 1              | 1.0          |

```typescript
import type { AudioBus, AudioBusConfig } from "@tokovo/core";

const busConfig: AudioBusConfig = {
  baseGain: 0.35,
  maxConcurrent: 1,
};
```

#### SoundCue — Active Sound Instance

Every playing sound is represented as a `SoundCue`:

```typescript
import type { SoundCue } from "@tokovo/core";

const cue: SoundCue = {
  soundId: "notification-chime",
  startFrame: 30,
  volume: 0.8,
  bus: "ui",
  priority: 50,
  loop: false,
  duration: 45,
  envelope: { attack: 3, release: 5 },
  duck: { targetBus: "music", amount: 0.5, attack: 5, release: 15 },
};
```

| Property     | Type            | Description                          |
| ------------ | --------------- | ------------------------------------ |
| `soundId`    | `string`        | ID registered in `registries.plugins.sounds` |
| `startFrame` | `number`        | Frame when sound started             |
| `volume`     | `number`        | Base volume (0-1)                    |
| `bus`        | `AudioBus`      | Target bus (music/ui/sfx/voice)      |
| `priority`   | `number`        | Higher = survives concurrency limits |
| `loop`       | `boolean`       | Loop continuously                    |
| `duration`   | `number`        | Length in frames (optional)          |
| `duck`       | `DuckRule`      | Auto-duck other buses                |
| `envelope`   | `AudioEnvelope` | Attack/release fade                  |

#### MusicBed — Background Music

Long-running background tracks use `MusicBed`:

```typescript
import type { MusicBed } from "@tokovo/core";

const bed: MusicBed = {
  id: "main-bgm",
  soundId: "upbeat-track",
  startFrame: 0,
  loop: true,
  baseGain: 0.35,
  moodTag: "romantic",
  crossfadeFrames: 60,
};
```

#### Crossfade Transitions

Smooth transitions between music tracks use `outgoingMusicBed`:

```typescript
// During crossfade, both beds exist:
audio: {
  musicBed: { soundId: "new-track", startFrame: 100, ... },
  outgoingMusicBed: { soundId: "old-track", fadeOutStart: 100, fadeOutDuration: 60 },
}
```

#### Policies — Deterministic Audio Rules

The policy system enforces consistent audio behavior:

```typescript
import { checkSpamPure, selectSoundsByPriority } from "@tokovo/core";

// Spam gate: prevent same sound repeated too fast
const result = checkSpamPure("click", frame, recentSounds, {
  spamGateFrames: 8,
  softVariant: "_soft",
});

// Concurrency: select highest-priority sounds per bus
const selected = selectSoundsByPriority(sounds, {
  maxConcurrentPerBus: { music: 1, ui: 3, sfx: 4, voice: 1 },
});
```

**Policy Rules:**

| Rule              | Default  | Description                   |
| ----------------- | -------- | ----------------------------- |
| Spam Gate         | 8 frames | Min frames between same sound |
| Soft Variant      | `_soft`  | Suffix for quieter repeat     |
| Music Concurrency | 1        | Only one music track          |
| Voice Concurrency | 1        | Only one voice-over           |
| UI Concurrency    | 3        | Max UI sounds                 |
| SFX Concurrency   | 4        | Max sound effects             |

#### Ducking — Automatic Level Management

When a sound with a `duck` rule plays, target buses are automatically attenuated:

```typescript
const voiceCue: SoundCue = {
  soundId: "narration",
  bus: "voice",
  duck: {
    targetBus: "music",
    amount: 0.7, // Duck music to 30%
    attack: 5, // 5-frame fade down
    release: 15, // 15-frame fade up after voice ends
  },
};
```

#### AudioLogger — Debug Audio Events

Enable runtime audio logging for debugging:

```typescript
import { AudioLogger } from "@tokovo/core";

// In browser console:
window.__TOKOVO_LOG_AUDIO = true;

// Logged events:
AudioLogger.play("notification", "ui", 30);
AudioLogger.stop("notification", 75);
AudioLogger.policyDrop({
  soundId: "click",
  bus: "ui",
  frame: 45,
  reason: "spam_gate",
  alternateSound: "click_soft",
});
AudioLogger.crossfade("old-track", "new-track", 60, 100);
```

#### Audio Event Handlers

The engine processes these audio runtime events:

| Event Type         | Handler             | Effect                                 |
| ------------------ | ------------------- | -------------------------------------- |
| `PLAY`             | `handlePlay()`      | Add SoundCue to activeSounds           |
| `STOP`             | `handleStop()`      | Remove by instanceId or soundId        |
| `STOP_ALL`         | `handleStopAll()`   | Clear bus or all sounds                |
| `FADE_OUT`         | `handleFadeOut()`   | Set fade target on sound               |
| `CROSSFADE`        | `handleCrossfade()` | Transition musicBed → outgoingMusicBed |
| `BACKGROUND_MUSIC` | `handleBGM()`       | Legacy BGM support                     |

### 4. Dependency Injection — Scoped Registries

Tokovo uses **scoped registries** per runtime. Create them once and pass them to
engine/replay/rendering calls:

```typescript
import { createTokovoRegistries } from "@tokovo/core";

const registries = createTokovoRegistries();

// Engine registries (reducers, middleware, lifecycle)
registries.engine;

// Plugin registries (apps, sounds, anchors, layouts, etc.)
registries.plugins;
```

### 5. Plugin System — Self-Contained Apps

Plugins are **batteries-included app modules** that bundle everything an app needs:

- **Reducer** — State update logic
- **Views** — React components
- **Sounds** — Audio assets
- **Layouts** — Platform-specific rendering
- **Anchors** — Semantic positioning
- **Notification adapters** — OS notification formatting

**Defining a plugin:**

```typescript
import { definePlugin } from "@tokovo/core";

export const whatsappPlugin = definePlugin({
  id: "app_whatsapp",
  displayName: "WhatsApp",

  // State reducer
  reducer: (draft, event) => {
    if (event.type === "MESSAGE_RECEIVED") {
      draft.appState.app_whatsapp.messages.push(event.payload.message);
    }
  },

  // UI component
  views: {
    AppRoot: WhatsAppView,
  },

  // Sound effects
  assets: {
    sounds: {
      notification: "/sounds/whatsapp-notification.mp3",
      sent: "/sounds/whatsapp-sent.mp3",
    },
  },

  // Layout strategy
  layouts: [
    {
      viewKind: "CHAT",
      platforms: ["ios"],
      computeLayout: (ctx) => ({
        /* layout state */
      }),
    },
  ],

  // Semantic anchors
  anchors: {
    providers: {
      send_button: (world, deviceId) => ({
        x: 0.85,
        y: 0.92,
        width: 0.1,
        height: 0.05,
      }),
    },
  },

  // Notification formatting
  notificationAdapter: {
    format: (notification) => ({
      title: notification.title,
      body: notification.body,
      icon: "/icons/whatsapp.png",
      color: "#25D366",
    }),
  },
});
```

**Registering plugins:**

```typescript
import { createTokovoRegistries, PluginManagerClass } from "@tokovo/core";

const registries = createTokovoRegistries();
const pluginManager = new PluginManagerClass(registries.plugins);

pluginManager.register(whatsappPlugin);
```

**Composing plugins:**

```typescript
import { composePlugin, defineReducer, defineViews } from "@tokovo/core";

const myPlugin = composePlugin(
  defineReducer({
    /* reducer */
  }),
  defineViews({ AppRoot: MyView }),
  defineLayouts([
    /* layouts */
  ]),
);
```

## Most-Used APIs

### `replay()` — Compute World State

```typescript
import { replay, createTokovoRegistries } from "@tokovo/core";

const registries = createTokovoRegistries();

const worldState = replay(initialWorld, events, currentFrame, {
  mode: "preview",
  fps: 30,
  registries: registries.engine,
});
```

### `definePlugin()` — Create Plugin

```typescript
import { definePlugin } from "@tokovo/core";

export const myPlugin = definePlugin({
  id: "app_myapp",
  displayName: "My App",
  reducer: (draft, event) => {
    /* ... */
  },
  views: { AppRoot: MyView },
});
```

### `PluginManagerClass.register()` — Register Plugin

```typescript
import { createTokovoRegistries, PluginManagerClass } from "@tokovo/core";

const registries = createTokovoRegistries();
const pluginManager = new PluginManagerClass(registries.plugins);

pluginManager.register(myPlugin);
```

### `prepareEpisode()` — Validate and Prepare

```typescript
import { prepareEpisode } from "@tokovo/core";

const prepared = prepareEpisode(episodeJSON);
// Returns: { initial, events, metadata, warnings }
```

### `resolveAnchor()` — Get Semantic Position

```typescript
import { createTokovoRegistries } from "@tokovo/core";

const registries = createTokovoRegistries();
const bounds = registries.plugins.anchors.resolveAnchor(
  "app_whatsapp:send_button",
  world,
  "main_phone",
);
// Returns: { x: 365, y: 850, width: 43, height: 43 }
```

## Determinism Guarantee

`@tokovo/core` guarantees **bit-for-bit determinism**:

> Given the same `initial` state, `events` array, and frame number `t`, the `replay()` function will **always** produce identical output.

**What this means:**

- ✅ Reproducible renders across machines
- ✅ AI-generated episodes render predictably
- ✅ Time-travel debugging (rewind/fast-forward)
- ✅ State snapshots for performance optimization

**How it's enforced:**

- ❌ No `Math.random()` — Use seeded RNG
- ❌ No `Date.now()` — Use `os.clock` from DeviceOSState
- ❌ No network calls — All data in events
- ❌ No external state — Pure functions only
